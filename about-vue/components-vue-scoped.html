Scoped-стили во Vue

Scoped-стили - это способ изолировать CSS-правила внутри компонентов Vue, предотвращая конфликты и обеспечивая чистоту кода.

Как работают scoped-стили?

Когда вы добавляете атрибут scoped к тегу <style> в вашем компоненте Vue, Vue автоматически генерирует уникальные атрибуты для всех элементов внутри этого компонента и соответствующих CSS-правил. Это гарантирует, что стили применяются только к этому конкретному компоненту и не влияют на другие части приложения.

Пример 1:

Компонент с scoped-стилями:

<template>
  <div class="box">Привет! 🎨</div>
</template>
<style scoped>
.box {
  background-color: lightblue;
  padding: 10px;
  border-radius: 5px;
}
</style>

Пример 2:

Компонент без scoped-стилей:

<template>
  <div class="box">Привет! 🎨</div>
</template>
<style>
.box {
  background-color: lightblue;
  padding: 10px;
  border-radius: 5px;
}
</style>
В первом примере стиль применяется только к этому компоненту, а во втором - ко всем элементам с классом .box в приложении.

Глубокие селекторы

Глубокие селекторы - это способ стилизовать вложенные компоненты или элементы внутри компонента, когда используются scoped-стили. В Vue 3 для этого используется специальный селектор ::v-deep.
Он позволяет "пробиться" через изоляцию scoped-стилей и применить стили к внутренним элементам дочерних компонентов.

::v-deep - это псевдо-селектор, который позволяет применять стили к элементам внутри дочерних компонентов, несмотря на то, что они находятся в области действия scoped-стилей родительского компонента.
систаксис следующий: ::v-deep(selector) или просто ::v-deep перед селектором.

Пример использования ::v-deep (Vue 3)

<style scoped>
::v-deep(.child) {
  color: red;
}
</style>
Объяснение кода:
  • В этом примере мы используем ::v-deep для стилизации элемента с классом .child, который находится внутри дочернего компонента.
  • Даже если дочерний компонент имеет свои собственные scoped-стили, правило внутри ::v-deep будет применено к элементу .child.
  • Это полезно, когда вам нужно изменить стили внутренних элементов, не нарушая изоляцию scoped-стилей.

🚀 Вывод: используйте ::v-deep для стилизации внутренних элементов дочерних компонентов, когда это необходимо.

Динамические стили через :style

:style - это директива Vue, которая позволяет привязывать стили к элементам динамически, используя объект или строку. Это особенно полезно, когда стили зависят от состояния компонента или данных.

Что нужно знать о :style:

  • Можно передавать объект с CSS-свойствами и их значениями.
  • Значения могут быть реактивными, что позволяет изменять стили в ответ на действия пользователя или изменения данных.
  • Когда использовать :style:
    • Динамическое изменение цветов, размеров, отступов и других стилей
    • Анимации и переходы
    • Темизация компонентов

    • Пример: динамическое изменение фона

    В этом примере мы создадим компонент, который позволяет пользователю менять цвет фона блока при нажатии на кнопку.

    <div id="dynamic-style-demo">
  <div class="box" :style="{ backgroundColor: bgColor }">
    Динамический цвет! 🎨
  </div>
  <button @click="changeColor">🔄 Сменить цвет</button>
</div>
    Объяснение кода:
    • Мы используем директиву :style для привязки свойства backgroundColor к реактивному свойству bgColor.
    • При нажатии на кнопку вызывается метод changeColor, который изменяет значение bgColor, что приводит к обновлению стиля фона блока.

    🚀 Вывод: используйте :style для динамического изменения стилей на основе состояния компонента.

Динамическое подключение CSS-классов через :class

:class - это директива Vue, которая позволяет динамически добавлять или удалять CSS-классы на основе состояния компонента или данных. Это особенно полезно для управления визуальными состояниями элементов, такими как активные, выделенные или ошибки.

Что нужно знать о :class:

  • Можно передавать объект, где ключи - это имена классов, а значения - булевы выражения, определяющие, должен ли класс быть применён.
  • Можно также передавать массив классов для добавления нескольких классов одновременно.
  • Когда использовать :class:
    • Переключение между состояниями (например, активный/неактивный)
    • Применение тем или стилей на основе условий
    • Управление видимостью элементов через классы

Пример: переключение темы

В этом примере мы создадим компонент, который позволяет пользователю переключаться между тёмной и светлой темами с помощью динамического класса.

<div id="dynamic-class-demo">
  <div :class="{ dark: isDark, light: !isDark }">Тема: {{ isDark ? "Тёмная" : "Светлая" }}</div>
  <button @click="isDark = !isDark">Переключить тему</button>
</div>
Объяснение кода:
  • Мы используем директиву :class для динамического применения классов dark и light на основе значения реактивного свойства isDark.
  • При нажатии на кнопку значение isDark переключается, что приводит к изменению применяемого класса и, соответственно, темы компонента.
🚀 Вывод: используйте :class для динамического управления классами на основе состояния компонента.

Практика: мини-приложение со scoped-стилями и динамикой

<div id="scoped-demo">
  <div :class="{ 'box': true, dark: isDark }" :style="{ borderColor: borderColor }">
    Цвет рамки и тема меняются!
  </div>
  <button @click="toggleTheme">Сменить тему</button>
  <button @click="changeBorder">Сменить цвет рамки</button>
</div>
Объяснение кода:
  • Мы используем директиву :class для динамического применения класса dark на основе значения реактивного свойства isDark.
  • Также используем директиву :style для динамического изменения цвета рамки на основе значения реактивного свойства borderColor.
  • При нажатии на кнопки вызываются методы toggleTheme и changeBorder, которые изменяют соответствующие свойства, что приводит к обновлению стилей компонента.
Проверьте работу
В этом примере применяется сразу и :class , и :style — вы можете менять тему и цвет рамки независимо!

Советы и частые ошибки

  • Не забывайте ставить scoped в <style> — иначе стили будут глобальными!
  • Для глубокого стилизации используйте только ::v-deep в новых проектах
  • :class — для переключения классов, :style — для уникальных значений
  • Старайтесь не смешивать глобальные и scoped-стили без необходимости
Частая проблема: забыли scoped — и ваши стили вдруг "разъехались" по всему проекту!

Лучшие практики работы со стилями во Vue

  • Всегда используйте scoped-стили для компонентов, если не требуется глобальная тема
  • Глобальные стили — только для сброса, шрифтов и корневых переменных
  • Старайтесь не использовать ::v-deep без крайней необходимости
  • Документируйте динамические классы и стили, чтобы избежать путаницы
Совет: Делайте компоненты максимально изолированными — это залог предсказуемости и красоты UI.

Реальные примеры использования scoped-стилей и динамики во Vue

Пример 1. Анимированная полоска прогресса

В этом примере мы создадим прогресс-бар, ширина которого изменяется динамически при нажатии на кнопку.

<div id="progress-demo">
  <div class="progress-bar" :style="{ width: percent + '%' }"></div>
  <button @click="inc">+10%</button>
</div>
Объяснение кода:
  • Мы используем директиву :style для привязки свойства width к реактивному свойству percent, которое определяет текущий процент заполнения прогресс-бара.
  • При нажатии на кнопку вызывается метод inc, который увеличивает значение percent на 10, что приводит к обновлению ширины прогресс-бара.

Пример 2. Стилизация дочернего компонента через ::v-deep

В этом примере родительский компонент использует ::v-deep для изменения стилей кнопки, которая находится внутри дочернего компонента.

<style scoped>
::v-deep(.child-btn) {
  background: orange;
  color: #fff;
}
</style>
<button class="child-btn">Кнопка</button>
Объяснение кода:
  • Мы используем селектор ::v-deep(.child-btn) для стилизации кнопки с классом .child-btn, которая находится внутри дочернего компонента.
  • Даже если дочерний компонент имеет свои собственные scoped-стили, правило внутри ::v-deep будет применено к кнопке.
