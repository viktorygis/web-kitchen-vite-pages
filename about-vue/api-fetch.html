Работа с Fetch API

Fetch API — (Fetch - перевод "получить") это встроенный инструмент в JavaScript, с помощью которого ты можешь отправлять HTTP-запросы на сервер, а также получать и отправлять данные ("по сети").
Он заменяет старый XMLHttpRequest ( XMLHttpRequest - "объект для запросов по XML" , устаревший метод для работы с сетью в JavaScript) и делает работу с сетью проще и удобнее благодаря использованию промисов (Promises - "обещания", современный способ работы с асинхронным кодом в JavaScript).

Fetch API делает работу с данными простой: ты можешь без лишних библиотек получать, изменять или удалять информацию на сервере.

С помощью Fetch API ты можешь:

  • 🔄 Получать данные для профиля, чата или списка товаров
  • 📝 Отправлять новую информацию (регистрация, заказ, комментарий)
  • 📦 Менять или удалять объекты на сервере
  • 🛠️ Обрабатывать ответы и ошибки для стабильной работы приложения

Fetch API — основной способ общаться с сервером в современных front-end приложениях!

🌐 Как работает Fetch API?

Fetch API — это метод JavaScript, который возвращает promise (обещание) в ответ на запрос, чтобы обработать данные или ошибки.

Как работает Fetch API?

  1. Отправляешь запрос через fetch(url)
  2. Получаешь ответ promise, из которого можно сделать JSON
  3. В зависимости от ответа сервера обновляешь интерфейс приложения

Пример простого GET-запроса:

fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data)) // Выводим данные в консоль
  .catch(error => console.error('Ошибка:', error));
Объяснение кода:
  • fetch('URL') — отправляет запрос на указанный URL
  • fetch — это функция, которая возвращает promise
  • promise — это обещание, что в будущем будет результат (успех или ошибка)
  • где здесь promise — это результат вызова fetch()
  • .then(response => response.json()) — преобразует ответ в JSON
  • .then — это метод для обработки успешного ответа
  • .then(data => ...) — работает с полученными данными
  • .catch(error => ...) — обрабатывает ошибки, если что-то пошло не так

️Совет: Всегда обрабатывай ошибки с помощью catch() для надёжной работы приложения!

🔎 GET запрос

GET — это метод HTTP-запроса, который используется для получения данных с сервера. Когда ты делаешь GET-запрос, ты запрашиваешь информацию, не изменяя её на сервере.
Это самый распространённый тип запроса, который используется для загрузки страниц, профилей пользователей, списков товаров и многого другого.
GET-запросы обычно не содержат тела запроса (body), так как они предназначены только для получения данных.
Испоьзуется по умолчанию, если не указан другой метод.

Где используется GET?

  • Загрузка профиля пользователя
  • Получение списка товаров в интернет-магазине
  • Загрузка комментариев к статье
  • Получение данных для отображения на странице

Пример использования GET-запроса с Fetch API:

fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data)) // Выводим данные в консоль
  .catch(error => console.error('Ошибка:', error));
Объяснение кода:
  • fetch('URL') — отправляет GET-запрос на указанный URL
  • response.json() — преобразует ответ в формат JSON
  • data — содержит полученные данные с сервера
  • console.log(data) — выводит данные в консоль для проверки
  • catch(error) — обрабатывает возможные ошибки при запросе

️Совет: GET-запросы не должны изменять данные на сервере — они только получают информацию!

🔎 POST запросы

POST — это метод HTTP-запроса, который используется для отправки данных на сервер. В отличие от GET-запросов, которые только получают информацию, POST-запросы позволяют создавать новые записи или изменять существующие данные на сервере.
Обычно POST-запросы используются при отправке форм, регистрации пользователей, создании заказов и других действиях, которые требуют передачи данных на сервер.

Где используется POST?

  • Регистрация нового пользователя
  • Добавление комментария к статье
  • Создание нового заказа в интернет-магазине
  • Отправка формы обратной связи

Пример POST запроса с Fetch API:

fetch('https://api.example.com/data', {
  method: 'POST', // Указываем, что это POST-запрос
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ name: 'John', age: 30 }) // Отправляем данные в формате JSON
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Ошибка:', error));
Объяснение кода:
  • method: 'POST' — указывает, что это POST-запрос
  • headers — задаёт заголовки запроса (например, тип контента)
  • body: JSON.stringify(...) — преобразует объект в строку JSON для отправки на сервер
  • Остальная часть кода аналогична GET-запросу

️Совет: Всегда проверяй, что данные отправляются в правильном формате (обычно JSON) при использовании POST-запросов!

Возможные параметры в POST запросах:

  • method — указывает метод запроса (в данном случае 'POST')
  • headers — задаёт заголовки запроса, например, тип контента
  • body — содержит данные, которые отправляются на сервер (обычно в формате JSON)

🔄 PUT запросы

PUT — это метод HTTP-запроса, который используется для обновления существующих данных на сервере. В отличие от POST-запросов, которые создают новые записи, PUT-запросы заменяют все данные указанного ресурса новыми данными.
Обычно PUT-запросы используются при изменении информации о пользователях, обновлении заказов и других действиях, которые требуют замены существующих данных на сервере.

Где используется PUT?

  • Обновление профиля пользователя
  • Изменение информации о заказе
  • Редактирование комментария
  • Обновление настроек приложения

Пример PUT запроса с Fetch API:

fetch('https://api.example.com/user/1', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ name: 'Alice', age: 25 })
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Ошибка:', error));
Объяснение кода:
  • method: 'PUT' — указывает, что это PUT-запрос
  • headers — задаёт заголовки запроса (например, тип контента)
  • body: JSON.stringify(...) — преобразует объект в строку JSON для отправки на сервер
  • Остальная часть кода аналогична GET и POST-запросам

️Совет: Используй PUT-запросы, когда нужно полностью заменить существующие данные на сервере!

Возможные параметры в PUT запросах:

  • method — указывает метод запроса (в данном случае 'PUT')
  • headers — задаёт заголовки запроса, например, тип контента
  • body — содержит данные, которые отправляются на сервер (обычно в формате JSON)

🔄 DELETE запросы

DELETE — это метод HTTP-запроса, который используется для удаления данных с сервера. В отличие от GET, POST и PUT, DELETE-запросы не возвращают данные, а просто удаляют указанный ресурс.

Где используется DELETE?

  • Удаление пользователя
  • Удаление комментария
  • Удаление заказа
  • Удаление файла или изображения

Пример DELETE запроса с Fetch API:

fetch('https://api.example.com/user/1', {
  method: 'DELETE'
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Ошибка:', error));
Объяснение кода:
  • method: 'DELETE' — указывает, что это DELETE-запрос
  • Остальная часть кода аналогична GET, POST и PUT-запросам

️Совет: Будь осторожен с DELETE-запросами — они удаляют данные без возможности восстановления!

Возможные параметры в DELETE запросах:

  • method — указывает метод запроса (в данном случае 'DELETE')

🛠 Обработка ответов и ошибок

Обработка ответов и ошибок — важный аспект работы с Fetch API. Даже если запрос выполнен успешно, сервер может вернуть ошибку (например, 404 или 500). Поэтому важно проверять статус ответа и обрабатывать ошибки корректно.
Это помогает создавать надёжные приложения, которые могут справляться с различными ситуациями.

Для обработки ответов и ошибок с Fetch API, можно использовать следующие подходы:

  1. Проверка статуса ответа с помощью response.ok
  2. Обработка ошибок с помощью catch()
  3. Логирование ошибок для отладки
  4. Отображение сообщений пользователю в случае ошибок
  5. Повторные попытки запросов при временных ошибках

Чтобы обработать ошибки, нужно:

  • Проверить, успешен ли ответ с помощью response.ok
  • Если ответ не успешен, выбросить ошибку с помощью throw new Error()
    throw new Error() - это способ выбросить ошибку в JavaScript, переводиться как "выбросить ошибку"
  • Использовать catch() для обработки любых ошибок (сетевых и наших)

Варианты ошибок:

  • Сетевые ошибки (например, отсутствие интернета)
  • Ошибки сервера (например, 500 Internal Server Error)
  • Ошибки клиента (например, 404 Not Found)

Пример обработки ответов и ошибок с Fetch API:

fetch(url)
  .then(response => {
    if (!response.ok) { // Проверяем, успешен ли запрос
      throw new Error(`Ошибка: ${response.status}`);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.error('Что-то пошло не так:', error));
  • response.ok — true, если HTTP-код 200-299
  • throw new Error() — выбрасывает ошибку, если запрос неудачен
  • catch() — ловит любые ошибки (сетевые и наши)

Совет: Всегда обрабатывай ошибки для создания надёжных приложений, которые могут справляться с различными ситуациями!

  • fetch - это встроенный метод JavaScript для выполнения HTTP-запросов к серверу. Используется для всех типов HTTP-запросов (GET, POST, PUT, DELETE).
  • По умолчанию используется метод GET, если не указано иное.
  • fetch('url') - это основной способ вызова API с помощью Fetch API. Внутри указывается URL, к которому нужно выполнить запрос.
  • fetch('url', {...}) - расширенный вызов, где можно указать дополнительные параметры, такие как метод запроса, заголовки и тело запроса (body).
  • method — указывает метод запроса (для POST, PUT, DELETE, GET)
  • headers — задаёт заголовки запроса, например, тип контента, для POST и PUT запросов (для DELETE и GET запросов обычно не нужен так как они не отправляют тело запроса)

  • response.json() — преобразует ответ сервера из формата JSON в объект JavaScript. Используется для всех типов запросов, когда ожидается ответ в формате JSON.
  • .then(data => console.log(data)) — обрабатывает полученные данные после успешного запроса. Используется для всех типов запросов.

Пример полного запроса POST с отработкой ошибок (реальный сервис)

Для демонстрации мы используем бесплатный сервис JSONPlaceholder, который отлично подходит для тренировок с запросами. 

const resultDiv = document.getElementById('fetch-post-result');
resultDiv.textContent = 'Выполнение запроса...';
fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    title: 'Hello World',
    body: 'Текст сообщения',
    userId: 1
  })
})
  .then(response => {
    if (!response.ok) {
      throw new Error(`Ошибка: ${response.status}`);
    }
    return response.json();
  })
  .then(data => {
    resultDiv.innerHTML = "Ответ сервера:
" +
      JSON.stringify(data, null, 2) + "
";
  })
  .catch(error => {
    resultDiv.innerHTML = "Что-то пошло не так: " + error.message + "";
  });
Объяснение кода:
  • https://jsonplaceholder.typicode.com/posts — адрес тестового API для добавления поста
  • Перед выполнением запроса выводится текст "Выполнение запроса..." ниже кнопки
  • method: 'POST' — указывает, что это POST-запрос
  • headers — задаёт необходимые заголовки запроса
  • body: JSON.stringify(...) — отправляем данные в формате JSON
  • if (!response.ok) { ... } — проверяем, что сервер вернул успешный ответ
  • throw new Error(...) — выбрасываем ошибку, если ответ не успешный
  • Результат (успех или ошибка) выводится в тот же блок под кнопкой
