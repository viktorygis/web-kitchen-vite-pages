Управление состоянием данных во Vuex: работа с API, загрузка и обновление

Управление состоянием данных во Vuex - это ключевой аспект разработки приложений на Vue.js. В него входит получение данных с сервера через Fetch API, сохранение этих данных в централизованном хранилище Vuex, а также обновление состояния приложения и отображение данных в компонентах Vue.

Основа Управление состоянием данных во Vuex

  • Понимание Vuex и его роли в управлении состоянием приложения
  • Использование Fetch API для получения данных с сервера
  • Сохранение и обновление данных в Vuex store
  • Обработка ошибок при работе с асинхронными операциями
  • Связывание компонентов Vue с Vuex для реактивного обновления интерфейса

Что такое Vuex и state?

Vuex — официальная библиотека для управления состоянием в приложениях на Vue.js. Она помогает централизованно хранить и изменять данные, что особенно полезно при работе с асинхронными операциями, такими как загрузка данных с сервера через Fetch API.

Для чего нужен Vuex?

  • Централизованное хранение данных приложения
  • Упрощённое управление состоянием при взаимодействии с сервером
  • Обеспечение реактивности компонентов при изменении данных
  • Упрощение отладки и тестирования

Что такое state?

State — это объект, который хранит все данные приложения в Vuex. Компоненты Vue могут подписываться на изменения state и автоматически обновляться при его изменении.

Как работает Vuex

Установка Vuex:

Сначала установи Vuex через npm или yarn:

npm install vuex --save
Интеграция Vuex в Vue-приложение:

Импортируй и подключи Vuex в главный файл приложения (обычно main.js):

// main.js
import Vue from 'vue';
import Vuex from 'vuex';
import App from './App.vue';
Vue.use(Vuex);
const store = new Vuex.Store({
  // Определение state, mutations, actions и getters
});
new Vue({ store, render: h => h(App) }).$mount('#app');
Создание хранилища Vuex:

Создай файл store.js для определения структуры хранилища:

// store.js
import Vue from 'vue';
import Vuex from 'vuex';
Vue.use(Vuex);
export default new Vuex.Store({
  state: {
    // Определение начального состояния
  },
  mutations: {
    // Определение мутаций для изменения состояния
  },
  actions: {
    // Определение действий для асинхронных операций
  },
  getters: {
    // Определение геттеров для получения данных из состояния
  }
});
Основные концепции Vuex:
  • State — централизованное хранилище данных приложения
  • Mutations — синхронные функции для изменения состояния
  • Actions — асинхронные функции для выполнения операций, таких как запросы к API
  • Getters — функции для получения и вычисления данных из состояния

Как работает Vuex с Fetch API?

  • Actions используют Fetch API для получения данных с сервера
  • После получения данных, actions вызывают mutations для обновления state
  • Компоненты Vue подписываются на изменения state и автоматически обновляются при изменении данных

Пример простейшего store с именем пользователя:

Создадим Vuex store, который хранит имя пользователя и позволяет его изменять:

// store.js
const store = new Vuex.Store({
  state: {
    userName: 'Петя'
  },
  mutations: {
    changeName(state, newName) {
      state.userName = newName;
    }
  }
});
Объяснение кода:
  • state — место для хранения данных
  • mutations — строго описывают, как можно менять state

Как получить данные из store в компоненте?

Чтобы использовать данные из Vuex store в компоненте Vue, можно обратиться к состоянию через this.$store.state:

// В компоненте
computed: {
  userName() {
    return this.$store.state.userName;
  }
}
Объяснение кода:
  • Используем this.$store.state для доступа к данным из store
  • Компонент автоматически обновится при изменении userName в store

🚀 Вывод: Vuex упрощает управление состоянием данных в приложениях Vue, особенно при работе с асинхронными операциями, такими как Fetch API. Он обеспечивает централизованное хранение данных и реактивное обновление компонентов.

🔗 Получение данных с сервера и сохранение в Vuex

Получение данных с сервера и сохранение в Vuex — это распространённый сценарий в разработке приложений на Vue.js. Обычно мы используем Fetch API для загрузки данных и сохраняем их в Vuex store для централизованного управления состоянием.

Как это работает?

Основные шаги для получения данных с сервера и сохранения их в Vuex:

  • Создание action в Vuex для выполнения Fetch-запроса
  • Обработка ответа и передача данных в mutation с помощью commit(commit - вызов mutation)
  • Обновление state с помощью mutation
  • Подписка компонентов на изменения state для отображения данных с помощью mapState

Зачем использовать Vuex для загруженных данных?

  • Можем обращаться к этим данным из любого компонента
  • Перерисовка всех «подписанных» компонентов при обновлении данных
  • Управляем загрузкой и ошибками централизованно

Пример fetch-запроса и обновления state:

Создадим Vuex store с action для загрузки списка пользователей с сервера и сохранения их в state:

// store.js
export default {
  state: {
    users: []
  },
  mutations: {
    setUsers(state, users) {
      state.users = users;
    }
  },
  actions: {
    loadUsers({ commit }) {
      fetch('https://jsonplaceholder.typicode.com/users')
        .then(response => response.json())
        .then(data => commit('setUsers', data))
        .catch(error => console.error('Ошибка при загрузке:', error));
    }
  }
}
Объяснение кода:
  • state.users — массив для хранения загруженных пользователей
  • mutation setUsers — обновляет state с полученными данными
  • action loadUsers — выполняет fetch-запрос, получает данные и вызывает mutation для обновления state
  • Обработка ошибок через catch

Внимание! Ниже мы разберём, как сделать загрузку асинхронной и более современной с помощью async/await.

⏳ Асинхронные actions: async/await + обработка ошибок

Асинхронные actions в Vuex позволяют выполнять операции, которые занимают время, такие как запросы к API. Использование async/await делает код чище и проще для понимания, а также облегчает обработку ошибок.

Почему async/await?

  • Код выглядит как синхронный, что упрощает чтение и понимание
  • Легче обрабатывать ошибки с помощью try...catch
  • Избегаем «ад колбеков» и цепочек .then()

Пример асинхронного action:

Перепишем предыдущий пример с использованием async/await:

// store.js
export default {
  actions: {
    async loadUsers({ commit }) {
      try {
        const response = await fetch('https://jsonplaceholder.typicode.com/users');
        const data = await response.json();
        commit('setUsers', data);
      } catch (error) {
        console.log('Ошибка при загрузке данных:', error);
      }
    }
  },
  mutations: {
    setUsers(state, users) {
      state.users = users;
    }
  },
  state: {
    users: []
  }
}
Объяснение кода:
  • async — функция делает асинхронную работу (например, fetch-запросы)
  • await — ждём ответа от сервера, прежде чем идти дальше
  • Всё, что после await, будет выполнено только после получения данных
  • Обработка ошибок через try...catch

🚀 Вывод: Использование async/await в Vuex actions упрощает работу с асинхронными операциями и улучшает читаемость кода. Это особенно полезно при взаимодействии с API и обработке ошибок.

🔗 Как компонент узнаёт о новых данных из store?

Связь компонента и Vuex store позволяет компонентам автоматически обновляться при изменении данных в хранилище. Это достигается с помощью специальных методов Vuex, таких как mapState и dispatch.

Как это работает?

Когда компонент вызывает action через dispatch, Vuex выполняет асинхронную операцию и обновляет state. Затем, благодаря mapState, компонент автоматически получает новые данные из store.

Компонент подписывается на изменения в store через mapState, поэтому при изменении данных в store, компонент перерисовывается автоматически.

Чтобы получить загруженные пользователи в компоненте, используй mapState и вызови action при запуске компонента (mount): 

// В компоненте
import { mapState } from 'vuex';

export default {
  mounted() {
    this.$store.dispatch('loadUsers'); // вызывать action при монтировании
  },
  computed: {
    ...mapState({
      users: state => state.users
    })
  }
}
Объяснение кода:
  • dispatch — вызывает асинхронный action
  • mapState — позволяет подписаться на часть state и использовать её как computed-свойство
  • Компонент сам обновится, когда users в store поменяются

🚀 Вывод: Связь компонентов Vue с Vuex store через mapState и dispatch обеспечивает реактивное обновление интерфейса при изменении данных, что упрощает разработку и улучшает пользовательский опыт.

🔥 Практика: загрузи пользователей через Vuex 4 и Vue 3!

Реальный пример использования Vuex для загрузки и отображения списка пользователей с сервера.
Мы создадим Vuex store с асинхронным action для получения данных через Fetch API и компонент для отображения этих данных.

Нужно будет создать 3 файла и связать их вместе:

  • store.js — определение Vuex store с state, mutations и actions
  • UsersList.vue — компонент для отображения списка пользователей
  • main.js — главный файл приложения для подключения Vuex store

Шаг 1: Создай Vuex store (Vue 3)

Создай файл store.js и определи state, mutations и actions для загрузки пользователей:

// store.js
import { createStore } from 'vuex'

export default createStore({
  state: {
    users: []
  },
  mutations: {
    setUsers(state, users) {
      state.users = users
    }
  },
  actions: {
    async loadUsers({ commit }) {
      try {
        const response = await fetch('https://jsonplaceholder.typicode.com/users')
        const data = await response.json()
        commit('setUsers', data)
      } catch (error) {
        console.log('Ошибка при загрузке данных:', error)
      }
    }
  }
})
Объяснение кода:
  • state.users — массив для хранения загруженных пользователей
  • mutation setUsers — обновляет state с полученными данными
  • action loadUsers — выполняет fetch-запрос, получает данные и вызывает mutation для обновления state
  • Обработка ошибок через catch

Шаг 2: Создай компонент для отображения пользователей

Создай компонент UsersList.vue, который будет отображать список пользователей из Vuex store:

// UsersList.vue
<template>
  <div>
    <h2>Пользователи</h2>
    <ul>
      <li v-for="user in users" :key="user.id">
        {{ user.name }}
      </li>
    </ul>
  </div>
</template>

<script>
import { mapState } from 'vuex'

export default {
  name: 'UsersList',
  computed: {
    ...mapState({
      users: state => state.users
    })
  },
  mounted() {
    this.$store.dispatch('loadUsers')
  }
}
</script>
Объяснение кода:
  • Используем mapState для получения списка пользователей из Vuex store
  • Вызываем action loadUsers при монтировании компонента для загрузки данных
  • Отображаем список пользователей с помощью директивы v-for

Шаг 3: Подключи store и компонент к приложению

Импортируй Vuex store в главный файл приложения и зарегистрируй компонент UsersList:

// main.js
import { createApp } from 'vue'
import App from './App.vue'
import store from './store'

createApp(App)
  .use(store)
  .mount('#app')
Объяснение кода:
  • Импортируем Vuex store и подключаем его к приложению с помощью .use(store)
  • Теперь все компоненты приложения смогут обращаться к Vuex store

Шаг 4: Запусти приложение и наслаждайся!

Теперь при запуске приложения компонент UsersList загрузит список пользователей с сервера и отобразит их.
