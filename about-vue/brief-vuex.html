Pinia (новый VueX) — современное управление состоянием во Vue

Pinia — переводиться как "сумка с деньгами", (ранее известная как VueX Next) это официальная библиотека для глобального управления состоянием в приложениях Vue 3. Она проще, модульнее и типобезопаснее старого VueX.
Для чего нужен Pinia?
Pinia позволяет централизовать и структурированно хранить состояние, а также организовать общение между компонентами на любом уровне вложенности. 

// Пример простого стора Pinia
import { defineStore } from 'pinia'
export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  actions: {
    increment() { this.count++ }
  }
})

Достоинства Pinia:

  • 🚀 Простая архитектура, максимально нативная для Vue 3 (Composition API).
  • 🧩 Гибкая модульность: каждый store — отдельный файл, легко масштабировать.
  • 🔌 Отличная интеграция с TypeScript.
  • ♻️ Горячее обновление, Devtools и SSR-ready.
  • 📝 Асинхронные действия и computed (getters) реализованы привычно для Vue-разработчика.

Структура Pinia в проекте

Pinia использует концепцию store — модульных "хранилищ" состояния.
Обычно все сторасы размещают в папке /stores или /pinia. Каждый store — это отдельный файл с функцией defineStore.

Пример структуры проекта с Pinia:

📂 src/
 ├── 📂 stores/
 │    ├── counter.js
 │    ├── user.js
 ├── App.vue
 ├── main.js
Описание ключевых элементов:
  • stores/ — директория для отдельных store-модулей.
  • counter.js — пример стора для счетчика.
  • user.js — store для пользователя (логин, роль и т.д.).
  • main.js — создание и подключение хранилища Pinia.

Использование Pinia в компоненте

Pinia интегрируется с приложением Vue через app.use(createPinia()).
Используйте сторасы во всех компонентах с помощью функции use*! Данные всегда реактивны.

Установка Pinia:

Если вы используете npm:

npm install pinia --save

Пример подключения Pinia:

В файле main.js:

// main.js
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'

const app = createApp(App)
app.use(createPinia())
app.mount('#app')

Пример стора:

Создание простого стора счетчика в stores/counter.js:

// stores/counter.js
import { defineStore } from 'pinia'
export const useCounterStore = defineStore('counter', {
  state: () => ({
    count: 0
  }),
  actions: {
    increment() {
      this.count++
    }
  }
})

Использование стора в компоненте:

В любом компоненте Vue:

<script setup>
import { useCounterStore } from '@/stores/counter'
const counter = useCounterStore()
</script>

<template>
  <div>
    {{ counter.count }}
    <button @click="counter.increment">+1</button>
  </div>
</template>

Actions и мутации

Actions — методы внутри стора, которые могут менять состояние или выполнять асинхронные операции (например, запросы к API).
Pinia не требует отдельного понятия "mutations", все изменения состояния делаются через actions или напрямую (state — реактивный).

Примеры действий (actions):

// stores/user.js
import { defineStore } from 'pinia'
export const useUserStore = defineStore('user', {
  state: () => ({
    name: '',
    isLoading: false
  }),
  actions: {
    async login(username) {
      this.isLoading = true
      // имитация запроса
      await new Promise(r => setTimeout(r, 1000))
      this.name = username
      this.isLoading = false
    }
  }
})
В отличие от VueX:
  • Нет отдельных mutation-файлов.
  • Можно изменять состояние прямо внутри action-функций.
  • actions могут быть как синхронными, так и асинхронными.

Getters (вычисляемые значения)

Getters — это вычисляемые свойства Pinia-стора, аналог computed в компонентах.
С помощью getters можно удобно собирать производные данные.

Пример использования getter:

// stores/counter.js
export const useCounterStore = defineStore('counter', {
  state: () => ({
    count: 0
  }),
  getters: {
    double: (state) => state.count * 2
  }
})

Использование getter в компоненте:

<template>
  <div>
    Удвоенное значение: {{ counter.double }}
  </div>
</template>

Модули (store) и организация Pinia

Pinia поощряет раздельную логику: один store — один файл.
Используйте разные store для разных частей состояния (например: counter, user, cart). Модули изолированы, но могут взаимодействовать друг с другом.

Пример нескольких store:

// stores/user.js
export const useUserStore = defineStore('user', {
  state: () => ({ name: '' })
})

// stores/counter.js
export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 })
})
Вы можете импортировать и использовать несколько store в одном компоненте:
<script setup>
import { useUserStore } from '@/stores/user'
import { useCounterStore } from '@/stores/counter'

const user = useUserStore()
const counter = useCounterStore()
</script>

Такой подход облегчает поддержку и масштабирование приложения.

Практика: счетчик на Pinia

Попробуйте реализовать простой счетчик с помощью Pinia — заведите store и управляйте состоянием в компоненте.
Ниже — полностью рабочий пример с Pinia через CDN. 

// stores/counter.js
import { defineStore } from 'pinia'
export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  actions: {
    increment() { this.count++ }
  }
})
// В компоненте:
<script setup>
import { useCounterStore } from '@/stores/counter'
const counter = useCounterStore()
</script>
<template>
  <div>
    {{ counter.count }}
    <button @click="counter.increment">+1</button>
  </div>
</template>

Лучшие практики работы с Pinia

Следуйте этим рекомендациям для чистого и масштабируемого кода в Pinia:

  • Используйте отдельную папку stores и отдельный файл для каждого стора.
  • Давайте сторам и getter-методам осмысленные имена.
  • Логику, которая относится к нескольким сторам, выносите в composables или сервисы.
  • Избегайте “гигантских” монолитных сторасов — каждый store должен быть независимым модулем.
  • Покрывайте бизнес-логику actions тестами.
  • Для сложных computed-вычислений используйте getters.
  • Интегрируйте Pinia Devtools для удобной отладки и анализа состояния.
  • Используйте TypeScript для типизации стора (особенно в крупных командах).
