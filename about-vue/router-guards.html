Навигационные охранники в Vue Router

Навигационные охранники (navigation guards) — это специальные методы, которые позволяют контролировать переходы между страницами в вашем приложении.
Они незаменимы, если нужно:

  • Ограничить доступ к отдельным маршрутам (например, только для авторизованных или админов)
  • Обновлять данные компонента при изменении параметров маршрута
  • Предупреждать пользователя о несохранённых изменениях перед уходом

Что изучаем?

  • beforeRouteEnter — проверка доступа перед входом
  • beforeRouteUpdate — реакция на смену параметров маршрута
  • beforeRouteLeave — предупреждение перед уходом
  • Глобальные guard'ы в router.js

beforeRouteEnter: запретить вход на страницу

Используется, чтобы допускать на страницу только определённых пользователей (например, только авторизованных).

Пример: только для авторизованных


export default {
  beforeRouteEnter(to, from, next) {
    if (!localStorage.getItem("user")) {
      next("/login"); // 🚷 Нет пользователя — отправляем на логин
    } else {
      next(); // ✅ Доступ разрешён!
    }
  }
}

to — маршрут, куда хотим перейти
from — маршрут, откуда пришли
next() — разрешить (next()) или перенаправить (next('/login'))

Если проверки не пройдены, пользователь не попадёт на защищённую страницу!

beforeRouteUpdate: актуализация при смене параметров

Иногда компонент не перезагружается, если меняется только параметр маршрута (например, /profile/1 → /profile/2). Для обновления данных используем beforeRouteUpdate.

Пример: подгрузка профиля по новому ID


export default {
  beforeRouteUpdate(to, from, next) {
    this.userId = to.params.id; // 🔄 Обновить ID
    next();
  }
}
  • to.params.id — новый ID из маршрута
  • this.userId = to.params.id — обновляем данные
  • next() — продолжаем переход
beforeRouteUpdate вызывается только при изменении параметров маршрута без полной перезагрузки!

beforeRouteLeave: предупредить о несохранённых изменениях

Позволяет показать предупреждение, если пользователь пытается уйти со страницы с незавершённой формой или несохранёнными изменениями.

Пример: подтверждение при уходе с заполненной формы


export default {
  beforeRouteLeave(to, from, next) {
    if (this.isFormDirty) {
      const answer = confirm("У вас есть несохранённые данные. Вы уверены, что хотите уйти?");
      if (!answer) return; // 🛑 Остаться на странице
    }
    next(); // ✅ Уйти
  }
}
  • this.isFormDirty — флаг несохранённых изменений
  • confirm() — подтверждение от пользователя
  • Не вызываем next() — переход отменён
Теперь случайные клики не приведут к потере важных данных!

Глобальные guards в router.js

Чтобы не дублировать проверки в каждом компоненте, можно настроить общий guard для группы маршрутов (например, для /admin).

Пример: защита всех страниц /admin


router.beforeEach((to, from, next) => {
  if (to.path.startsWith("/admin") && !localStorage.getItem("admin")) {
    next("/login"); // 🚫 Не админ — на логин!
  } else {
    next(); // ✅ Доступ разрешён
  }
});
  • to.path.startsWith("/admin") — проверяем, что идём в админку
  • Нет admin в localStorage — не пускаем
Глобальные guards делают защиту всего раздела быстрой и централизованной!

Практика: навигационные охранники

  • Реализуй beforeRouteEnter для страницы профиля: пускать только авторизованных
  • Обновляй данные профиля через beforeRouteUpdate, если меняется ID
  • На странице формы — beforeRouteLeave предупреждает о несохранённых изменениях
  • Настрой глобальный guard для раздела /admin

App.vue

<!-- App.vue -->
<template>
  <div>
    <router-link to="/">Главная</router-link>
    <router-link :to="{ name: 'profile', params: { id: 1 } }">Профиль 1</router-link>
    <router-link :to="{ name: 'profile', params: { id: 2 } }">Профиль 2</router-link>
    <router-link to="/form">Форма</router-link>
    <router-link to="/admin">Админка</router-link>
    <button @click="loginUser">Войти как user</button>
    <button @click="loginAdmin">Войти как admin</button>
    <button @click="logout">Выйти</button>
    <router-view/>
  </div>
</template>

<script>
export default {
  methods: {
    loginUser() { localStorage.setItem("user", "yes"); localStorage.removeItem("admin"); alert("Вошли как user!"); },
    loginAdmin() { localStorage.setItem("user", "yes"); localStorage.setItem("admin", "yes"); alert("Вошли как admin!"); },
    logout() { localStorage.clear(); alert("Вышли из системы!"); }
  }
}
</script>

Profile.vue

<!-- Profile.vue -->
<template>
  <div>
    <h2>Профиль пользователя №{{ userId }}</h2>
  </div>
</template>

<script>
export default {
  data() { return { userId: this.$route.params.id }; },
  beforeRouteEnter(to, from, next) {
    if (!localStorage.getItem("user")) {
      next("/login");
    } else {
      next(vm => { vm.userId = to.params.id; });
    }
  },
  beforeRouteUpdate(to, from, next) {
    this.userId = to.params.id;
    next();
  }
}
</script>

Form.vue

<!-- Form.vue -->
<template>
  <div>
    <h2>Форма анкеты</h2>
    <input v-model="value" placeholder="Введите что-нибудь...">
    <button @click="save">Сохранить</button>
  </div>
</template>

<script>
export default {
  data() { return { value: "", isFormDirty: false }; },
  watch: { value() { this.isFormDirty = true; } },
  methods: { save() { this.isFormDirty = false; alert("Сохранено!"); } },
  beforeRouteLeave(to, from, next) {
    if (this.isFormDirty) {
      const answer = confirm("У вас есть несохранённые данные. Вы уверены, что хотите уйти?");
      if (!answer) return;
    }
    next();
  }
}
</script>

Частые ошибки и вопросы

  • Ошибка: Не вызвали next() — переход "зависает"!
  • Ошибка: Забыли учесть параметры маршрута в beforeRouteUpdate
  • Вопрос: Как защитить сразу все страницы /admin?
    Ответ: Глобальный guard в router.js
  • Вопрос: Как отменить переход?
    Ответ: Не вызывать next() или вызвать next(false)
Используй навигационные охранники для удобной и безопасной навигации!
