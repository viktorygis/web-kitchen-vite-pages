Слоты во Vue

Slots - это мощный механизм во Vue, позволяющий создавать гибкие и переиспользуемые компоненты. С помощью слотов вы можете передавать контент из родительского компонента в дочерний, делая компоненты более универсальными.

Виды слотов:

  • Базовые слоты - для передачи простого контента.
  • Именованные слоты - для передачи контента в определённые "кармашки".
  • Резервное содержимое (Fallback Content) - для отображения контента по умолчанию, если родитель ничего не передал.

🚀 Слоты делают ваши компоненты более гибкими и позволяют легко изменять их содержимое без необходимости модифицировать сам компонент.

Базовые слоты

Базовые слоты - это простой способ передать контент из родительского компонента в дочерний. Они позволяют вставлять произвольный HTML или другие компоненты внутрь компонента-обёртки.

Как использовать базовые слоты:

  • В дочернем компоненте используйте тег <slot></slot> в том месте, где хотите вставить контент из родителя.
  • В родительском компоненте просто поместите нужный контент между тегами дочернего компонента.

Пример базового слота:

<!-- Родительский компонент -->
<div id="app">
  <container-component>
    <p>Содержимое, которое можно менять без проблем! 😊</p>
  </container-component>
</div>

<script>
Vue.createApp({})
  .component("container-component", {
    template: `
      <div style="border:2px dashed #42b983;padding:10px;">
        <slot></slot>
      </div>
    `
  })
  .mount("#app");
</script>
Объяснение кода:
  • <container-component> - это дочерний компонент, который будет содержать слот.
  • Внутри <container-component> в шаблоне используется тег <slot></slot>, который обозначает место для вставки контента из родителя.
  • В родительском компоненте между тегами <container-component> и </container-component> помещается параграф с текстом. Этот текст будет вставлен в место слота внутри дочернего компонента.

🚀 Базовые слоты позволяют легко вставлять динамический контент в компоненты, делая их более гибкими и переиспользуемыми!

Именованные слоты

Именованные слоты - это расширение базовых слотов, позволяющее создавать несколько слотов внутри одного компонента. Каждый слот имеет своё имя, что позволяет передавать контент в определённые "кармашки" компонента.

Как использовать именованные слоты:

  • В дочернем компоненте используйте тег <slot name="имя_слота"></slot> для каждого именованного слота.
  • В родительском компоненте используйте тег <template #имя_слота>...</template> для передачи контента в соответствующий слот.

Виды именованных слотов:

  • Обычные именованные слоты - позволяют передавать контент в определённые зоны компонента.
  • Дефолтный слот - используется для контента, не попавшего ни в один именованный слот.

Пример именованных слотов:

<!-- multi-slot-card - Родительский компонент -->
<multi-slot-card>
  <!-- Передача контента в именованные слоты -->
  <!-- #header - слот для заголовка -->
  <template #header>
    <h2>Заголовок 🌟</h2>
  </template>

  <!-- #footer - слот для футера -->
  <template #footer>
    <small>Футер</small>
  </template>

  <p>Основной текст карточки.</p>
</multi-slot-card>

<!-- multi-slot-card.vue - Дочерний компонент -->
<template>
  <!-- Компонент с несколькими именованными слотами -->
  <div class="card">
  <!-- Именованный слот для заголовка -->
    <slot name="header"></slot>
    <!-- Дефолтный слот -->
    <slot>По умолчанию, если ничего не пришло</slot>
    <!-- Именованный слот для футера -->
    <slot name="footer"></slot>
  </div>
</template>
Объяснение кода:
  • <multi-slot-card> - это дочерний компонент с несколькими именованными слотами.
  • В шаблоне дочернего компонента используются три слота: <slot name="header">, <slot> (дефолтный слот) и <slot name="footer">.
  • В родительском компоненте используются теги <template #header> и <template #footer> для передачи контента в соответствующие слоты.
  • Контент, не попавший ни в один именованный слот, будет вставлен в дефолтный слот.

🚀 Именованные слоты позволяют создавать сложные и гибкие компоненты с несколькими зонами для динамического контента!

Резервное содержимое

Резервное содержимое - (Fallback Content - перевод "запасной контент") это контент, который отображается внутри слота, если родительский компонент не передал никакого контента. Это полезно для обеспечения базового отображения компонента, когда нет данных от родителя.

Как использовать резервное содержимое:

  • В дочернем компоненте внутри тега <slot></slot> добавьте HTML-контент, который должен отображаться по умолчанию.
  • Если родительский компонент не передаст контент, будет отображено резервное содержимое.

Пример резервного содержимого:

<!-- Родительский компонент -->
<div id="app">
  <!-- Нет переданного контента -->
  <container-component></container-component>
</div>

<script>
Vue.createApp({})
  .component("container-component", {
    template: `
      <div style="border:2px dashed #42b983; padding:10px;">
       <!-- Резервное содержимое слота -->
        <slot>Тут пока пусто 👀</slot>
      </div>
    `
  })
  .mount("#app");
</script>
Объяснение кода:
  • В дочернем компоненте <container-component> внутри тега <slot></slot> добавлено резервное содержимое: Тут пока пусто 👀.

🚀 Резервное содержимое обеспечивает базовое отображение компонента, когда родитель не передал контент, улучшая пользовательский опыт!

Множественные слоты: комплексные компоненты

Множественные слоты - это использование нескольких слотов внутри одного компонента для создания более сложных и гибких интерфейсов. Это особенно полезно для компонентов, которые требуют различных областей контента, таких как карточки, модальные окна и панели.

Как использовать множественные слоты:

  • В дочернем компоненте определите несколько слотов с разными именами с помощью тега <slot name="имя_слота"></slot>.
  • В родительском компоненте используйте теги <template #имя_слота>...</template> для передачи контента в соответствующие слоты.

Пример множественных слотов:

<multi-slot-card>
  <template #header>
    <h2>Заголовок 🌟</h2>
  </template>
  <p>Основной текст карточки.</p>
  <template #footer>
    <small>Футер</small>
  </template>
</multi-slot-card>

<!-- multi-slot-card.vue -->
<template>
  <div class="card">
    <slot name="header"></slot>
    <slot>По умолчанию, если ничего не пришло</slot>
    <slot name="footer"></slot>
  </div>
</template>
Объяснение кода:
  • В дочернем компоненте <multi-slot-card> определены три слота: <slot name="header">, <slot> (дефолтный слот) и <slot name="footer">.
  • В родительском компоненте используются теги <template #header> и <template #footer> для передачи контента в соответствующие слоты.
  • Контент, не попавший ни в один именованный слот, будет вставлен в дефолтный слот.
🚀 Множественные слоты позволяют создавать сложные и гибкие компоненты с несколькими зонами для динамического контента, улучшая структуру и переиспользуемость кода!
