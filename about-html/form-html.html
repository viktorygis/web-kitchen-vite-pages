HTML: Формы и элементы ввода

Формы являются основным способом взаимодействия пользователя с веб-сайтом.
Они позволяют собирать данные, отправлять их на сервер или использовать для различных операций, таких как поиск, регистрация или авторизация.

На этой странице обсуждаются основные элементы HTML-форм, такие как <input> , <textarea> , <select> и другие.

Базовая структура формы

Каждая HTML-форма начинается с тега <form> . Основные атрибуты:

  • action : URL-адрес, на который отправляются данные формы
  • method : метод отправки данных ( GET или POST )
action — адрес, куда отправляются данные после нажатия на кнопку.
method : GET — данные видны в адресной строке, подходит для поиска. POST — данные скрыты, лучше для регистрации/логина. 

<form action="submit.php" method="POST">
  <!-- Элементы формы -->
  <input type="text" placeholder="Ваше имя">
  <button>Отправить</button>
</form>

Тег <input>

<input> — универсальный элемент для ввода данных. Тип ввода указывается с помощью атрибута type .

Текстовые поля - text

Это самый базовый тип <input>
Они создаются с помощью type="text" и позволяют пользователю вводить любой текст, например, имя, комментарий или адрес. 

<input type="text" placeholder="Ваше имя">

Что происходит в коде?

  • <input> — создаёт поле ввода.
  • type="text" — делает это поле текстовым, куда можно вводить любые буквы или цифры.
  • placeholder="Ваше имя" — добавляет подсказку внутри поля, которая исчезнет, когда пользователь начнёт вводить текст.

⚠️ Важно: input — одиночный тег, он не требует закрывающего, а также без атрибута type поле не будет работать корректно.

⚡ Текстовые поля — это база. Ты будешь использовать их почти на каждой странице!

Пример

<input type="text" placeholder="Ваше имя">

Текстовые поля - email

Поля с type="email" позволяют вводить электронную почту. 

<input type="email" placeholder="Ваш email">

⚠️ Важно: Поля email автоматически проверяют, введён ли адрес правильно (например, есть ли символ @).
Это облегчает жизнь пользователям и уменьшает количество ошибок.

Пример

<input type="email" placeholder="Ваш email">

Текстовые поля - password

Поля с type="password" поле для ввода пароля (символы скрыты). 

<input type="password" placeholder="Введите пароль">

⚠️ Важно: Всё, что пользователь вводит, превращается в звёздочки или точки. ✨
Это делает поле безопасным для ввода пароля (но сервер тоже должен быть защищён, не забывай об этом!).

Пример


<input type="password" placeholder="Введите пароль">

Текстовые поля - checkbox

Поля с type="checkbox" флажок, позволяют выбирать один или несколько вариантов.
Это как маленькая коробочка 📦, в которую можно поставить галочку. 


<input type="checkbox" id="fly">
<label for="fly">Полёты</label>

<input type="checkbox" id="invisibility">
<label for="invisibility">Невидимость</label>

Что происходит в коде?

  • <input type="checkbox"> — это сам чекбокс, маленький квадрат, который можно отметить.
  • <label> — это подпись для чекбокса. Она объясняет пользователю, что он выбирает.
  • for в теге <label> связывает текст с конкретным чекбоксом через его id .
  • Когда пользователь кликает по тексту внутри <label> , связанный чекбокс автоматически отмечается.

⚠️ Важно: всегда используйте атрибут for в <label> и соответствующий id в <input> — это сделает форму удобнее и доступнее.

☑️ Чекбоксы отлично подходят для выбора нескольких вариантов из списка!

Пример


<input type="checkbox" id="news">
<label for="news">Получать новости</label>

<input type="checkbox" id="offers">
<label for="offers">Специальные предложения</label>

<input type="checkbox" id="events">
<label for="events">Уведомления о событиях</label>

Переключатели — radio

Поля с type="radio" позволяют выбрать только один вариант из группы.
Это как круглый переключатель ⭕, куда можно поставить только одну точку. 


<input type="radio" id="male" name="gender" value="male">
<label for="male">Мужской</label>

<input type="radio" id="female" name="gender" value="female">
<label for="female">Женский</label>

<input type="radio" id="other" name="gender" value="other">
<label for="other">Другое</label>

Что происходит в коде?

  • <input type="radio"> — это сам радиобаттон (переключатель), круглый элемент, который можно выбрать.
  • name — одинаковое значение у всех радио в группе. Благодаря этому можно выбрать только один вариант из нескольких.
  • value — значение, которое отправится на сервер при выборе этого варианта.
  • <label> — подпись для переключателя, объясняет, что выбирает пользователь.
  • for в теге <label> связывает текст с конкретным переключателем через его id .
  • Когда пользователь кликает по тексту <label> , соответствующий переключатель выбирается.
  • Если не выбран ни один radio-кнопка, никакое значение из этой группы не отправится на сервер.
  • Можно задать один из radio как выбранный по умолчанию с помощью атрибута checked .

⚠️ Важно: все радио-кнопки, относящиеся к одной группе выбора, должны иметь одинаковое name .
Это обеспечивает корректную работу выбора только одного варианта.

🔘 Радио-переключатели используют для случаев, когда из вариантов можно выбрать только один!

Пример


<input type="radio" id="visa" name="payment">
<label for="visa">Visa</label>

<input type="radio" id="mastercard" name="payment">
<label for="mastercard">MasterCard</label>

<input type="radio" id="mir" name="payment">
<label for="mir">МИР</label>

Числовые поля — number

Поля с type="number" позволяют вводить только числа.
Это удобно для возраста, количества, цены и других числовых значений. 


<input type="number" id="age" name="age" min="0" max="120">
<label for="age">Возраст</label>

<input type="number" id="quantity" name="quantity"  

min="1" max="10" value="1">
<label for="quantity">Количество</label>

Что происходит в коде?

  • <input type="number"> — это поле для ввода только числовых значений.
  • min и max — задают минимальное и максимальное допустимое значение.
  • value — устанавливает начальное значение по умолчанию.
  • step — определяет шаг изменения значения (например, step="0.01" для цены с копейками).
  • Пользователь может вводить число с клавиатуры или менять значение стрелками вверх/вниз.
  • При попытке ввести не-число (буквы) браузер не позволит отправить форму.
  • <label> — подпись для поля, объясняет что нужно ввести.
  • Для удобства на мобильных устройствах автоматически появляется цифровая клавиатура.

⚠️ Важно: используйте min и max для ограничения диапазона значений, а step — для дробных чисел.
Проверьте ввод и на стороне сервера!

🔢 Числовые поля подходят для возраста, количества, суммы заказа и других числовых данных!

Пример


<input type="number" id="price" name="price" min="0" step="0.01" value="0.00">
<label for="price">Стоимость (₽)</label>

<input type="number" id="year" name="year" min="1900" max="2100" value="2025">
<label for="year">Год</label>

Пример

Этот пример использует все основные типы <input> (text, email, password, number, checkbox, radio, date), а также <select> и <textarea>. 

<form action="/submit" method="post">
  <label for="name">Имя:</label>
  <input type="text" id="name" name="name" required>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="form-password">Пароль:</label>
  <input type="password" id="form-password" name="form-password" required>

  <label for="form-age">Возраст:</label>
  <input type="number" id="form-age" name="form-age" min="1" max="120">

  <label><input type="checkbox" name="subscribe"> Подписаться на новости</label>

  <label>Пол:</label>
  <input type="radio" id="male" name="gender" value="male">
  <label for="male">Мужской</label>
  <input type="radio" id="female" name="gender" value="female">
  <label for="female">Женский</label>
  <input type="radio" id="other" name="gender" value="other">
  <label for="other">Другое</label>

  <label for="country">Страна:</label>
  <select id="country" name="country">
    <option value="ru">Россия</option>
    <option value="us">США</option>
    <option value="de">Германия</option>
  </select>

  <label for="birthday">Дата рождения:</label>
  <input type="date" id="birthday" name="birthday">

  <label for="message">Комментарий:</label>
  <textarea id="message" name="message" rows="3"></textarea>

  <button type="submit">Отправить</button>
</form>

Тег <textarea>

<textarea> — элемент формы для ввода многострочного текста, в отличие от обычного <input> .
Используется для комментариев, отзывов, сообщений и других случаев, когда пользователю нужно ввести длинный текст.

Где применяют <textarea> ?

  • Комментарии 💬
  • Отзывы 📝
  • Сообщения
  • Описание профиля
  • Любой длинный текст

<textarea placeholder="Напишите ваш комментарий"></textarea>

rows и cols

🖋️ <textarea> можно настроить по размеру с помощью атрибутов rows (высота в строках) и cols (ширина в символах).

  • rows — количество видимых строк (по высоте).
  • cols — примерная ширина поля (в символах, не в пикселях).

Также можно управлять шириной через CSS (например, width: 100%; ), тогда cols не обязателен.

textarea rows="5" cols="30"


<textarea rows="5" cols="30" placeholder="Напишите ваш отзыв">
</textarea>

textarea rows="2" cols="10"


<textarea rows="2" cols="10" placeholder="Короткое сообщение">
</textarea>

Текст по умолчанию

Можно добавить текст между открывающим и закрывающим тегом <textarea> . Он сразу появится внутри поля как начальный текст. 


<textarea rows="5" cols="30">
Ваш текст по умолчанию
</textarea>

maxlength

Если нужно ограничить количество символов, используйте атрибут maxlength .
Например, чтобы оставить отзыв длиной не более 200 символов: 


<textarea maxlength="200"
placeholder="Ваш отзыв (до 200 символов)">
</textarea>

id и name

  • id — нужен для связи с <label> , чтобы при клике по подписи курсор попадал в поле.
  • name — обязательно для отправки значения на сервер. Без name данные не отправятся!
  • Пример разметки c подписью и кнопкой отправки:

<form action="/submit" method="post">
  <label for="message">Сообщение:</label>
  <textarea id="message" name="message" rows="4" cols="50">
  </textarea>

  <button type="submit">Отправить</button>
</form>

Дополнительные советы

  • Пользователь может растягивать <textarea> за уголок (если не отключать resize в CSS).
  • Для обязательных полей добавьте атрибут required .
  • Можно использовать placeholder для подсказки внутри поля.
  • Сделайте подписи <label> кликабельными — это удобно и для пользователей, и для доступности.

📝 Итого: <textarea> — универсальный элемент для ввода длинных сообщений, отзывов, комментариев и любых других текстов!

Тег <select> — выпадающий список

Тег <select> используется для создания выпадающего списка, из которого пользователь может выбрать один или несколько вариантов.

Базовый пример


  <form action="/submit" method="post">
    <label for="country">Выберите страну:</label>
    <select id="country" name="country">
      <option value="ru">Россия</option>
      <option value="us">США</option>
      <option value="de">Германия</option>
    </select>

    <button type="submit">Отправить</button>
  </form>

Что важно знать о <select>?

  • Каждый вариант определяется тегом <option> .
  • Атрибут value в <option> — это значение, которое отправляется на сервер при выборе этого пункта.
  • Можно задать вариант по умолчанию, добавив атрибут selected к нужной опции.
  • <label> с for и id делает список более доступным.
  • При отсутствии value отправляется текст опции.

Атрибут size — высота списка

Атрибут size позволяет задать количество одновременно видимых пунктов в списке.
Если size="1" (по умолчанию), список выглядит как выпадающий. Если указать большее значение, список станет "раскрытым" и покажет сразу несколько вариантов. 


<select name="language" size="3">
  <option>Английский</option>
  <option>Русский</option>
  <option>Китайский</option>
  <option>Немецкий</option>
  <option>Французский</option>
</select>

Примечание: Если используется multiple , рекомендуется явно указывать size для удобства выбора.

selected - Пример с выбранным вариантом по умолчанию

Можно задать вариант по умолчанию, добавив атрибут selected к нужной опции. 


  <select name="city" id="city">
    <option value="msk">Москва</option>
    <option value="spb" selected>Санкт-Петербург</option>
    <option value="ekb">Екатеринбург</option>
  </select>

disabled - Отключённые варианты

Можно сделать отдельные пункты недоступными для выбора с помощью disabled . 


  <select>
    <option>Основной вариант</option>
    <option disabled>Недоступный вариант</option>
    <option>Другой вариант</option>
  </select>

💡 Совет: Используй disabled, чтобы временно отключить варианты или показать, что опция недоступна. Это гораздо лучше, чем просто удалять пункт — пользователь хотя бы узнает, что он существовал. 😉

Множественный выбор — multiple

Можно разрешить выбирать сразу несколько вариантов с помощью атрибута multiple .
Для удобства такой список часто делают выше, с помощью атрибута size .

Теперь пользователь сможет выделить сразу несколько опций (например, зажав клавишу Ctrl или Command). 🎯


  <select name="hobbies" multiple size="4">
    <option>Чтение</option>
    <option>Спорт</option>
    <option>Музыка</option>
    <option>Путешествия</option>
  </select>

Группировка вариантов — <optgroup>

Для организации длинных списков можно использовать тег <optgroup> с подписью.

    <optgroup label="Ягоды">
      <option>Клубника</option>
      <option>Черника</option>

      <option>Апельсин</option>

По умолчанию <select> и <option> выглядят просто и зависят от браузера и операционной системы. Но с помощью CSS их можно сделать современнее.
Вот примеры базовой стилизации:

  • С помощью appearance: none скрывается стандартная стрелка браузера, чтобы использовать свою SVG-иконку.
  • Можно менять цвета, отступы, радиус скругления и фон.
  • Для кроссбраузерной стилизации используйте все варианты appearance .
  • Можно добавить эффект наведения и фокуса для лучшего UX.

💡 Для сложной стилизации и поиска часто используют готовые решения на JavaScript (например, Select2, Choices.js или библиотеку компонентов вашего фреймворка).

💡 Совет: Используйте <select> для выбора из ограниченного набора вариантов. Для длинных списков комбинируйте с поиском на JavaScript.

Теги <fieldset> и <legend>

Тег <fieldset> используется для группировки элементов формы.
Он визуально выделяет группу, обрамляя её рамкой.
Для описания группы можно использовать тег <legend> , который добавляет заголовок внутри рамки.

  • <fieldset> помогает:
    • Группировать элементы формы, чтобы было понятно, какие из них связаны.
    • Добавлять порядок и структуру в твой код (и в жизнь тоже 😅).
    • Делать формы более удобными для пользователей.
  • <legend> — это название группы <fieldset> . Оно помогает пользователю понять, что за данные собраны в этом разделе. Представь, что <legend> — это как ярлык на коробке 📦 с надписью "Личные данные". 

<form>

    <input type="text" id="username" name="username" required>

    <label for="password">Пароль:</label>
    <input type="password" id="password" name="password" required>

    <button type="submit">Войти</button>
  </fieldset>
</form>

Элементы даты и времени

HTML поддерживает специализированные типы ввода для даты и времени, которые делают ввод данных удобным и стандартизированным.
Такие поля автоматически показывают пользователю календарь или часы, адаптированные под его устройство и язык.
Это упрощает заполнение форм и снижает вероятность ошибок.

  • date : выбор конкретной даты (например, 2025-07-30)
  • time : выбор времени (например, 14:30)
  • datetime-local : выбор даты и времени без часового пояса (например, 2025-07-30T14:30)
  • month : выбор месяца и года (например, 2025-07)
  • week : выбор недели в году (например, 2025-W31)

Базовый пример


<form>
  <label for="appointment-date">Дата записи:</label>
  <input type="date" id="appointment-date" name="date">

  <label for="appointment-time">Время записи:</label>
  <input type="time" id="appointment-time" name="time">

  <label for="appointment-datetime">Дата и время записи:</label>
  <input type="datetime-local" id="appointment-datetime" name="datetime">

  <button type="submit">Забронировать</button>
</form>

Пример: выбор месяца и недели


<form>
  <label for="report-month">Отчетный месяц:</label>
  <input type="month" id="report-month" name="report-month">

  <label for="work-week">Рабочая неделя:</label>
  <input type="week" id="work-week" name="work-week">
</form>

Ограничения и значения по умолчанию

Для всех элементов можно задать минимальные и максимальные значения через min и max , а также значение по умолчанию через value .
Например: 


<input type="date" min="2025-08-01" max="2025-12-31" value="2025-09-01">
<input type="time" min="09:00" max="18:00" value="10:00">

Пояснения и советы

  • Такие поля отлично работают на мобильных устройствах: появляется удобный нативный календарь или часы.
  • Значения всегда отправляются в стандартизированном формате, удобном для обработки на сервере.
  • Можно комбинировать разные типы для сложных форм (например, бронирование, планирование событий, отчётность).
  • В некоторых старых браузерах поддержка может быть ограничена. Для сложных случаев используют сторонние JS-библиотеки.

💡 Совет: Всегда указывайте min и max для ограничения диапазона дат/времени, если это важно для вашей задачи!

Валидация форм

Валидация — это проверка данных, которые вводит пользователь в форму.
Если что-то заполнено некорректно, браузер аккуратно сообщит об ошибке и не даст отправить форму, пока пользователь не исправит данные.

HTML предоставляет встроенные средства валидации формы с помощью специальных атрибутов. Они позволяют без единой строчки JavaScript сразу проверять корректность ввода:

  • required — поле обязательно для заполнения.
  • min , max — минимальные и максимальные значения для числовых и дат.
  • pattern — регулярное выражение для проверки формата ввода (например, для телефона или индекса).
  • maxlength и minlength — ограничение длины текста в символах.
  • type — тип поля (например, email , number , tel ), который автоматически включает базовую валидацию.
  • step — шаг изменения для чисел и времени (например, только чётные числа, или время с шагом 15 минут).

required — обязательное поле

required — самый простой способ сделать поле обязательным.
Если пользователь оставит поле пустым, форма не отправится: браузер покажет подсказку, что поле нужно заполнить.

Пример:

<form>
  <input type="text" placeholder="Имя" required />
  <input type="email" placeholder="Email" required />
  <button type="submit">Отправить</button>
</form>

minlength и maxlength — длина текста

Чтобы сделать пароль надёжным, можно задать минимальную длину с помощью minlength .
Например: 


<form>
  <input type="password" placeholder="Придумайте пароль" minlength="8" required />
  <button type="submit">Создать аккаунт</button>
</form>

📏 Чтобы ограничить максимальное количество символов, используй атрибут maxlength :

min и max — диапазон значений

Для числовых или датных полей можно ограничить диапазон с помощью min и max .
Например, для возраста — только значения от 18 до 60: 


<form>
  <label for="age">Возраст (18-60):</label>
  <input type="number" id="age" name="age" min="18" max="60" required>

  <label for="date">Дата не раньше 2025-08-01:</label>
  <input type="date" id="date" name="date" min="2025-08-01">

  <button type="submit">Отправить</button>
</form>

pattern — проверка по шаблону

Атрибут pattern позволяет задать шаблон ввода с помощью регулярного выражения.
Например, проверка номера телефона: 


<form>
  <input type="tel" pattern="\+7\d{10}" placeholder="+7XXXXXXXXXX" required />
  <button type="submit">Проверить</button>
</form>

Если формат не совпадает, браузер покажет сообщение об ошибке.

step — шаг изменения

Атрибут step задаёт шаг (интервал) для числовых и временных полей.
Например, можно разрешить вводить только чётные числа: 


<form>
  <input type="number" min="0" max="100" step="2" />
</form>

Итоги и советы

  • Встроенная валидация работает автоматически во всех современных браузерах.
  • Поля с ошибками подсвечиваются, а браузер показывает подсказку.
  • Для сложных проверок можно добавить JavaScript , но всегда начинайте с HTML-валидаторов.
  • Делайте формы понятными: используйте placeholder и label для подсказок.

💡 Совет: Используйте как можно больше встроенных возможностей HTML — это просто, удобно и безопасно!

Советы по HTML-формам

  • Используйте атрибут for в <label> для привязки метки к полю ввода.
  • Добавляйте атрибут required для полей, которые обязательно должны быть заполнены.
  • Для больших форм группируйте поля с помощью <fieldset> и <legend> .
  • Подсказки для пользователя можно добавлять с помощью атрибута placeholder .
  • Проверьте форму на корректную обработку данных на стороне сервера.
  • Используйте type="email" , type="number" и другие специализированные типы для удобства пользователя и валидации.
  • Для мобильных устройств используйте подходящий inputmode (например, inputmode="numeric" ).
Совет: Актуальные HTML-формы делают сбор данных простым и удобным для пользователя!

Кнопка — button

Элемент <button> создаёт интерактивную кнопку на странице.
Обычно используется для отправки формы или запуска какого-то действия.
<button> Нажми на меня </button> 


<button type="submit">Отправить</button>
<button type="reset">Сбросить</button>
<button type="button">Просто кнопка</button>

Что происходит в коде?

  • <button> — создаёт кнопку.
  • Текст между открывающим и закрывающим тегом <button> — это надпись на кнопке.
  • Кнопку можно стилизовать с помощью CSS или добавить иконку внутрь.

⚠️ Важно: если не указать атрибут type , по умолчанию кнопка внутри формы будет submit и отправит форму.
Для кнопок, которые не должны отправлять форму, всегда указывайте type="button" .

🔘 <button> — универсальный элемент для любых действий пользователя!

Кнопка отправки формы — <button type="submit"> и <input type="submit">

Кнопка submit предназначена для отправки данных формы на сервер. 🚀
Такой кнопкой можно воспользоваться двумя способами:

  • С помощью тега <button> :
    <button type="submit">Отправить</button>
  • С помощью тега <input type="submit"> :
    <input type="submit" value="Я хочу пиццу! 🍕">

Для <input type="submit"> текст на кнопке задаётся через атрибут value .
Вместо стандартного "Submit" можно написать что-то креативное: 


<input type="submit" value="Я хочу пиццу! 🍕">

Для <button> текст и даже HTML-иконки (например, <span>🍕</span> ) можно размещать между открывающим и закрывающим тегом: 


<button type="submit">Отправить заказ 🍕</button>

Кнопка сброса формы — <button type="reset">

Кнопка reset используется для очистки всех полей формы — значения вернутся к исходным.
Это удобно, если пользователь хочет начать заполнение формы с нуля. 


<button type="reset">Сбросить</button>

Обычная кнопка — <button type="button">

Кнопка button не связана с отправкой или сбросом формы.
Она используется для любых пользовательских действий: открытия модальных окон, запуска скриптов, переключения интерфейса и так далее. 


<button type="button" onclick="alert('Привет!')">Просто кнопка</button>

Стилизация кнопок

Кнопки можно сделать ярче и красивее с помощью CSS.
Например, изменить цвет, форму, добавить эффекты наведения или тень.
Вот базовый пример оформления кнопки: 


<button class="my-btn">Яркая кнопка</button>

<style>
.my-btn {
  background-color: #1976d2;
  color: #fff;
  border: none;
  border-radius: 5px;
  padding: 10px 24px;
  font-size: 1em;
  cursor: pointer;
  transition: background 0.2s;
}
.my-btn:hover {
  background-color: #1565c0;
}
</style>

Что происходит в стилях?

  • background-color — задаёт цвет фона кнопки.
  • color — цвет текста.
  • border и border-radius — убирают стандартную рамку и скругляют углы.
  • padding — внутренние отступы, чтобы кнопка выглядела объёмнее.
  • font-size — размер шрифта на кнопке.
  • cursor: pointer — курсор в виде руки при наведении.
  • transition — плавное изменение цвета при наведении ( :hover ).

Краткое описание
