Базовые теги HTML

Базовые теги HTML — это основные строительные блоки веб-страниц, которые определяют структуру и содержание документа. Они позволяют браузеру правильно отображать текст, изображения, ссылки и другие элементы.

Знание базовых тегов HTML необходимо для создания и редактирования веб-страниц. Эти теги помогают организовать контент, сделать его доступным и понятным для пользователей и поисковых систем.

Пример базовых тегов HTML:

<!DOCTYPE html>
<html lang="ru">
<head>
   <meta charset="UTF-8">
   <title>Пример страницы</title>
   <link rel="stylesheet" href="styles.css">
   <script src="script.js"></script>
</head>
<body>
   <h1>Заголовок страницы</h1>
   <p>Это пример параграфа текста.</p>
   <a href="https://example.com">Ссылка на сайт</a>
   <img src="image.jpg" alt="Описание изображения">
</body>
</html>

Виды базовых тегов HTML

  • <html>

    <html> — Корневой элемент любого HTML-документа, который служит контейнером для всего содержимого страницы (включая <head> и <body>).

    Атрибут lang — указывает основной язык содержимого страницы (например, lang="ru" для русского). Это важно для доступности, озвучивания текста скринридерами и для поисковых систем.

    Атрибут xmlns (XML namespace) — применяется редко, обычно автоматически добавляется браузером или редакторами. Для обычных HTML-страниц указывать не обязательно.

    Тег <html> всегда единственный и самый верхний в документе. Он не может быть вложен в другие элементы и обязательно содержит внутри себя блоки <head> и <body> .

    Другие полезные атрибуты <html> :

    • dir — направление текста на странице ( ltr — слева направо, rtl — справа налево). Например: <html dir="rtl">
    • class — CSS-класс для стилизации всей страницы. Обычно используется редко на корневом элементе: <html class="main-html">
    • id — уникальный идентификатор, редко применяется к тегу <html> , но может быть полезен для скриптов или ссылок: <html id="top">
    • manifest — путь к файлу-оболочке для офлайн-приложений (устаревший, почти не используется).
    • data-* — пользовательские атрибуты для хранения дополнительных данных, например: <html data-theme="dark">
    Примечание: Тег <html> должен быть первым элементом в документе и не может быть вложен в другие теги.

    Пример:

    <html lang="ru">
  <head>...</head>
  <body>...</body>
</html>

  • <head>

    <head> — Элемент, содержащий метаданные о документе: заголовок страницы, кодировку, подключаемые стили, скрипты и другую служебную информацию.

    Содержимое <head> не отображается напрямую на странице, но влияет на работу браузера, SEO и внешний вид сайта.

    Основные теги внутри <head> :

    • <title> — задаёт заголовок страницы, который отображается во вкладке браузера и в поисковых системах.
    • <meta> — метаданные: например, <meta charset="UTF-8"> указывает кодировку, <meta name="description" content="..."> — описание для SEO.
    • <link> — подключает внешние ресурсы: CSS-файлы, фавиконки, шрифты.
    • <style> — внутренние CSS-стили для страницы.
    • <script> — подключение или размещение JavaScript-кода.
    • <base> — устанавливает базовый путь для относительных ссылок.
    Атрибуты <head> :
    • profile — (устарел) мог определять ссылку на профиль метаданных.
    • id — уникальный идентификатор, редко используется на этом элементе.
    • class — CSS-класс, обычно не применяется, но допустим.
    Примечание: Тег <head> всегда размещается после <html> и до <body> . Внутри <head> нельзя размещать элементы, предназначенные для отображения на странице.

    Пример:

    <head>
  <title>Документ</title>
  <meta charset="UTF-8">
  <link rel="stylesheet" href="style.css">
</head>

  • <title>

    <title> — Тег, который задаёт заголовок страницы. Этот заголовок отображается на вкладке браузера, в поисковых системах и когда добавляете страницу в закладки.

    <title> размещается только внутри <head> и должен быть уникальным для каждой страницы.

    Заголовок помогает пользователю понять, о чём страница, и влияет на SEO.

    Примечание: Можно использовать любые символы, но не стоит делать заголовок слишком длинным.

    Пример:

    <title>Моя первая страница</title>

  • <body>

    <body> — Основной контейнер для всего видимого содержимого страницы: текста, картинок, ссылок, списков, форм и других элементов.

    Внутри <body> размещаются все элементы, которые пользователь видит и с которыми взаимодействует.

    Атрибуты <body> :

    • onload — запускает JavaScript при загрузке страницы.
    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.
    • style — встроенные CSS-стили.

    Примечание: В документе всегда должен быть только один <body> .

    Пример:

    <body>
  <h2>Заголовок</h2>
  <p>Текст параграфа.</p>
</body>

  • <h1> - <h6>

    <h1> - <h6> — Теги для заголовков различных уровней. <h1> — главный заголовок страницы, <h2> — подзаголовок и так далее до <h6> .

    Используйте <h1> только один раз на странице — это важно для SEO. Остальные заголовки структурируют текст и помогают сделать контент удобным для восприятия.

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор, например для якорных ссылок.

    Примечание: Не пропускайте уровни: <h2> должен идти после <h1> и так далее.

    Пример:

    <h1>Заголовок 1 уровня</h1>
<h2>Заголовок 2 уровня</h2>

  • <p>

    <p> — Тег для создания параграфа текста. Каждый <p> автоматически отделяется от соседних параграфов отступом.

    Используйте для обычного текста, описаний, пояснений.

    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.
    • style — встроенные CSS-стили.

    Примечание: Внутри <p> нельзя размещать блочные элементы, только строчные (например, <a> , <span> ).

    Пример:

    <p>Это пример параграфа.</p>

  • <ol> и <ul>

    <ol> — нумерованный список, <ul> — маркированный (ненумерованный) список.

    Каждый элемент списка помещается в тег <li> . Списки можно вкладывать друг в друга.

    • type — тип маркера (например, type="A" для <ol> ).
    • start — начальный номер для <ol> .
    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Примечание: Для логической структуры используйте списки, а не просто переносы строк.

    Примеры:

    <ol>
  <li>Первый элемент</li>
  <li>Второй элемент</li>
</ol>

<ul>
  <li>Элемент списка</li>
  <li>Еще один элемент списка</li>
</ul>

  • <a>

    <a> — Тег для создания гиперссылок (перехода по адресу или якорю).

    Главный атрибут — href (адрес или якорь). Можно добавить target="_blank" для открытия ссылки в новой вкладке.

    • href — адрес назначения.
    • target — как открыть ссылку ( _blank , _self ).
    • rel — отношения и безопасность (например, rel="noopener" ).
    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Примечание: Можно использовать для ссылок на внешние, внутренние или якорные адреса.

    Пример:

    <a href="https://example.com">Перейти на пример</a>

  • <img>

    <img> — Тег для вставки изображений на страницу.

    Обязательный атрибут — src (путь к файлу изображения).

    • src — путь к изображению.
    • alt — текст для описания картинки (важен для доступности).
    • width и height — размеры картинки.
    • title — всплывающая подсказка при наведении.
    • class — CSS-класс для стилизации.
    • id — уникальный идентификатор.

    Примечание: Тег <img> — одиночный (нет закрывающего тега). Всегда указывайте alt для доступности!

    Пример:

    <img src="image.jpg" alt="Описание изображения">
