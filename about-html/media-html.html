HTML: Мультимедийные теги
(img, iframe, audio, video)

Мультимедийные теги HTML позволяют добавлять на страницу изображения, видео, аудио и сторонние ресурсы — делают сайт более наглядным и интерактивным.

Основные теги

В HTML есть несколько ключевых тегов для работы с мультимедиа.
Каждый из них имеет свои особенности и предназначение:

  • <img> — отображение изображений на странице.
  • <iframe> — встраивание сторонних сайтов, видео, карт, документов.
  • <audio> — воспроизведение аудиофайлов.
  • <video> — воспроизведение видеофайлов.
  • <embed> — встраивание медиа- и других файлов (PDF, SVG, Flash).
  • <object> — подключение различных объектов, с поддержкой альтернативного контента.
  • <canvas> — рисование графики на странице с помощью JavaScript.

<img>

Тег <img> — один из самых часто используемых в HTML.
Он позволяет вставлять изображения в контент, делая его более наглядным и привлекательным.
Важно всегда указывать атрибут alt — это не только помогает поисковым системам индексировать изображения, но и делает ваш сайт доступным для людей с нарушениями зрения.
Также старайтесь оптимизировать изображения для быстрой загрузки страниц. 
<img src="https://via.placeholder.com/300x200" alt="Пример" width="300" height="200">
  • Тег <img> вставляет изображение. Атрибут alt нужен для доступности и SEO.
  • Размеры можно задать атрибутами width и height .

Пример:

<picture>

Тег <picture> позволяет выводить разные изображения в зависимости от устройства, плотности пикселей или поддерживаемого формата. 

<picture>
  <source srcset="image.webp" type="image/webp">
  <source srcset="image.jpg" type="image/jpeg">
  <img src="image.jpg" alt="Адаптивное изображение">
</picture>

<figure> и <figcaption>

Теги <figure> и <figcaption> используются для группировки иллюстраций и их подписей. 

<figure>
  <img src="photo.jpg" alt="Фото">
  <figcaption>Подпись к фото</figcaption>
</figure>

<svg>

Тег <svg> позволяет создавать векторную графику прямо в HTML. 

<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>

<map> и <area>

Теги <map> и <area> позволяют создавать интерактивные области на изображениях (image maps).

Как это работает?

  • Тег <map> определяет область изображения, на которую можно кликнуть.
  • Тег <area> определяет интерактивные области внутри тега <map> .
  • Каждая область может иметь свою форму (прямоугольник, круг и т.д.) и ссылку.

Как визуально обозначить области?

Визуально области могут быть обозначены с помощью CSS, как показано ниже.

<img src="plan.png" usemap="#planmap" alt="План">
<map name="planmap">
  <area shape="rect" coords="34,44,270,350" href="room.html" alt="Комната">
  <area shape="circle" coords="130,120,40" href="garden.html" alt="Сад">
</map>

<iframe>

<iframe> - это HTML-тег, который используется для встраивания одного веб-документа (например, другой веб-страницы, видео, карты) внутрь текущей страницы. Он позволяет отображать содержимое с другого URL в рамках (frame) на вашей странице. 

<iframe src="https://www.youtube.com/embed/dQw4w9WgXcQ"
   width="400"
   height="225"
   title="Пример видео"
   allowfullscreen>
</iframe>
  • <iframe> позволяет встроить видео, карту или другой сайт.
  • Атрибут allowfullscreen даёт пользователю перейти в полноэкранный режим.
  • Атрибут sandbox ограничивает доступ встроенного контента к функциям браузера, повышая безопасность.
  • Некоторые сайты могут запрещать показ своего содержимого в <iframe> через специальные HTTP-заголовки.
  • Для доступности обязательно указывайте атрибут title .

Пример с sandbox:

<iframe
  src="https://www.example.com"
  width="400"
  height="225"
  sandbox="allow-scripts allow-same-origin"
  title="Пример с sandbox"></iframe>

Стилизация через CSS:

iframe {
  border-radius: 12px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
  border: none;
}

<audio>

<audio> - элемент позволяет воспроизводить звук на веб-странице.

В браузере

Пример с атрибутами:

<audio controls>
  <source src="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-1.mp3" type="audio/mpeg">
  Ваш браузер не поддерживает аудиоэлемент.
</audio>

Основные правила:

  • Тег <audio> нужен для воспроизведения звука.
  • <source> нужен для указания различных аудиофайлов и их форматов.
    Например, вы можете указать один файл в формате MP3 и другой в формате OGG.
  • Для доступности всегда добавляйте текст внутри тега на случай, если браузер не поддерживает <audio> .

Атрибуты:

  • src - указывает путь к аудиофайлу.
  • type - указывает MIME-тип аудиофайла. (значения: audio/mpeg - для MP3, audio/ogg - для OGG )
  • controls - отображает элементы управления воспроизведением.
  • autoplay - позволяет автоматически запускать звук при загрузке страницы.
  • loop - повторяет воспроизведение аудио по кругу.
  • muted - делает аудио изначально беззвучным.
  • preload - управляет загрузкой аудиофайла (значения: auto - браузер сам решает, загружать аудио или нет , metadata - загружает только метаданные (например, длительность) , none - не загружает аудио до тех пор, пока пользователь не нажмет кнопку воспроизведения ).

Пример с дополнительными атрибутами:

<audio
  controls                <!-- отображает элементы управления (play, pause, громкость и др.) -->
  autoplay                <!-- автоматически запускает аудио при загрузке страницы -->
  loop                    <!-- повторяет аудио по кругу -->
  muted                   <!-- аудио стартует без звука -->
  preload="auto"          <!-- стратегия предзагрузки аудио (auto, metadata, none) -->
>
  <source src="audio.mp3" type="audio/mpeg">      <!-- основной аудиофайл (mp3) -->
  <source src="audio.ogg" type="audio/ogg">      <!-- альтернативный формат для совместимости -->
  Ваш браузер не поддерживает аудиоэлемент.       <!-- текст для старых браузеров -->
</audio>

<video>

<video> — элемент позволяет воспроизводить видеоролики на веб-странице.

В браузере

Пример с атрибутами:

<video controls width="320" height="180" poster="https://via.placeholder.com/320x180">
  <source src="https://www.w3schools.com/html/mov_bbb.mp4" type="video/mp4">
  Ваш браузер не поддерживает видеоэлемент.
</video>

Основные правила:

  • Тег <video> нужен для воспроизведения видеофайлов.
  • <source> нужен для указания различных видеофайлов и их форматов.
    Например, можно указать файлы в формате MP4 и WEBM.
  • Для доступности всегда добавляйте текст внутри тега, если браузер не поддерживает <video> .

Атрибуты:

  • src — указывает путь к видеофайлу.
  • type — указывает MIME-тип видеофайла ( video/mp4 , video/webm и др.).
  • controls — отображает элементы управления: play, pause, громкость и пр.
  • autoplay — позволяет автоматически запускать видео при загрузке страницы.
  • loop — повторяет воспроизведение видео по кругу.
  • muted — делает видео изначально беззвучным.
  • preload — управляет загрузкой видеофайла ( auto — браузер сам решает, загружать видео или нет, metadata — загружает только метаданные, none — не загружает видео до нажатия play).
  • width и height — задают размеры видеоплеера.
  • poster — изображение-заставка, отображается до начала воспроизведения.

Пример с дополнительными атрибутами:

<video
  src="video.mp4"                <!-- путь к видеофайлу; можно использовать <source> вместо src -->
  controls                       <!-- отображает элементы управления (play, pause, громкость и др.) -->
  autoplay                       <!-- автоматически запускает видео при загрузке страницы -->
  loop                           <!-- повторяет видео по кругу -->
  muted                          <!-- видео стартует без звука -->
  preload="auto"                 <!-- стратегия предзагрузки видео (auto, metadata, none) -->
  width="640"                    <!-- ширина видеоплеера -->
  height="360"                   <!-- высота видеоплеера -->
  poster="preview.jpg"           <!-- изображение-заставка до начала воспроизведения -->
  playsinline                    <!-- воспроизведение внутри страницы (на мобильных устройствах) -->
  disablePictureInPicture        <!-- запрет режима "картинка в картинке" -->
>
  <source src="video.mp4" type="video/mp4">         <!-- основной видеофайл (mp4) -->
  <source src="video.webm" type="video/webm">       <!-- альтернативный формат для совместимости -->
  Ваш браузер не поддерживает видеоэлемент.          <!-- текст для старых браузеров -->
</video>

<track>

Тег <track> используется внутри <video> для добавления субтитров, титров и других текстовых дорожек. 

<video controls>
  <source src="movie.mp4" type="video/mp4">
  <track kind="subtitles" src="subtitles_ru.vtt" srclang="ru" label="Русский">
</video>

<embed>

Тег <embed> позволяет вставлять медиафайлы (PDF, SVG, видео, аудио) прямо в страницу.

Пример кода:

<embed
  src="presentation.pdf"         <!-- путь к файлу для вставки -->
  type="application/pdf"         <!-- MIME-тип файла (PDF, SVG, видео, аудио и др.) -->
  width="320"                    <!-- ширина области отображения -->
  height="180"                   <!-- высота области отображения -->
>
  • Тег <embed> позволяет вставлять медиафайлы (PDF, SVG, видео, аудио) прямо в страницу.
  • Поддерживает атрибуты src , type , width , height .
  • Не имеет "парного" закрывающего тега (самозакрывающийся).
  • Подходит для вставки интерактивных и мультимедийных файлов (например, PDF-презентаций или SVG-графики).
  • Для доступа к содержимому нет fallback-текста — если браузер не поддерживает <embed> , файл не будет показан.

Пример с SVG:

<embed
  src="image.svg"                <!-- путь к SVG-файлу -->
  type="image/svg+xml"           <!-- MIME-тип SVG -->
  width="320"                    <!-- ширина -->
  height="180"                   <!-- высота -->
>

Разница с другими тегами

  • Тег <embed> используется для встраивания медиафайлов, но не поддерживает fallback-контент и не имеет элементов управления воспроизведением.
  • В отличие от <video> и <audio> , которые предназначены для мультимедиа, тег <embed> более универсален и может использоваться для вставки различных типов файлов.
  • Тег <embed> не имеет "парного" закрывающего тега (самозакрывающийся).

Пример разницы между <embed> и <audio>

<object>

Тег <object> используется для встраивания различных типов мультимедиа и других ресурсов в HTML-документ.

Пример использования

Пример кода:

<object
  data="diagram.svg"             <!-- путь к файлу/ресурсу -->
  type="image/svg+xml"           <!-- MIME-тип файла -->
  width="320"                    <!-- ширина области отображения -->
  height="180"                   <!-- высота области отображения -->
>
  Не удалось загрузить изображение. <!-- fallback-контент -->
</object>

Основные правила:

  • Тег <object> используется для встраивания различных типов мультимедиа и других ресурсов в HTML-документ.
  • Тег <object> — универсальный для вставки SVG, PDF, HTML, Flash и других внешних ресурсов.

Атрибуты:

  • data — путь к файлу/ресурсу.
  • type — MIME-тип файла.
  • width — ширина области отображения.
  • height — высота области отображения.

Разница с другими тегами

  • <object> — более универсальный и поддерживает различные типы контента.
  • <embed> — используется для встраивания контента, но с меньшей гибкостью.
  • <iframe> — встраивает другой HTML-документ, но не поддерживает такие типы, как SVG или PDF.

Пример с PDF и fallback:

<object
  data="presentation.pdf"        <!-- путь к PDF-файлу -->
  type="application/pdf"         <!-- MIME-тип PDF -->
  width="320"                    <!-- ширина -->
  height="180"                   <!-- высота -->
>
  Ваш браузер не поддерживает отображение PDF.
  <a href="presentation.pdf">Скачать PDF</a>
</object>

Когда какой тег использовать?

  • Используйте <object> для встраивания различных типов мультимедиа и других ресурсов.
  • Используйте <embed> для встраивания контента, когда вам не нужна гибкость <object> .
  • Используйте <iframe> для встраивания другого HTML-документа.

<canvas>

Тег <canvas> используется для рисования графики с помощью JavaScript.

canvas — это инструмент для рисования в браузере. Его используют для всего, где нужна динамика и визуализация: графики, игры, анимация, редакторы, интерактивные элементы, обработка изображений и даже просто для рисования на сайте!

Пример использования:

Пример кода:

HTML:
<canvas
  id="demoCanvas"                <!-- уникальный идентификатор для JS -->
  width="320"                    <!-- ширина области canvas -->
  height="180"                   <!-- высота области canvas -->
>
  Ваш браузер не поддерживает canvas.
</canvas>
JavaScript:
<script>
  // Получаем элемент canvas по id
  const canvas = document.getElementById("demoCanvas");

  // Проверяем, поддерживает ли браузер canvas
  if (canvas.getContext) {
    // Получаем 2D-контекст для рисования
    const ctx = canvas.getContext("2d");

    // Задаём цвет заливки и рисуем прямоугольник
    ctx.fillStyle = "skyblue";
    ctx.fillRect(20, 40, 100, 60);

    // Меняем цвет и рисуем текст
    ctx.fillStyle = "navy";
    ctx.font = "18px sans-serif";
    ctx.fillText("Canvas demo", 30, 80);
  }
</script>

Основные правила:

  • Тег <canvas> должен иметь заданные width и height в атрибутах, а не в CSS.
  • Для рисования используйте JavaScript и методы getContext() и drawImage() .
  • Не забывайте проверять поддержку <canvas> в браузере.
  • Внутри тега можно указать текст для браузеров, не поддерживающих <canvas> .

Атрибуты:

  • id - уникальный идентификатор для элемента, используется в JavaScript.
  • width и height - задают размеры области рисования.
  • style - позволяет задавать стили для элемента canvas.

Примеры: progress и meter

Элемент <progress>

Элемент <progress> отображает прогресс выполнения задачи.
Задайте текущий прогресс через атрибут value , максимальное значение — через max .

  • Используется для отображения загрузки, выполнения процессов и т.д.
<progress value="70" max="100">70%</progress>

В результате:

70%

Элемент <meter>

Элемент <meter> отображает значение в заданном диапазоне.
Часто используется для визуализации уровня заряда, оценки результатов, скорости и т.п.

  • value : текущее значение.
  • min : минимальное значение (по умолчанию — 0).
  • max : максимальное значение (по умолчанию — 1).
  • low : низкий порог (значение ниже считается низким).
  • high : высокий порог (значение выше считается высоким).
  • optimum : оптимальное значение. 
<meter value="0.6" min="0" max="1" low="0.4" high="0.8" optimum="0.7">60%</meter>

В результате:

60%

В чем разница?

Элементы <progress> и <meter> имеют разные семантические значения и предназначены для разных целей.

  • <progress> используется для отображения прогресса выполнения задачи (например, загрузка файла).
  • <meter> отображает измеренное значение в пределах заданного диапазона (например, уровень заряда батареи).

Пример разницы

<progress value="70" max="100">70%</progress>
<meter value="0.6" min="0" max="1" low="0.4" high="0.8" optimum="0.7">60%</meter>

Пример: Нажмите на элементы

70% 60%

Практика

Попробуйте взаимодействовать с элементами: воспроизвести аудио или видео с помощью кнопки.

Советы и типичные ошибки

  • Указывайте alt для изображений — это важно для доступности.
  • Добавляйте несколько форматов в <audio> и <video> для поддержки разных браузеров.
  • Проверяйте размер медиафайлов — большие изображения и видео замедляют загрузку сайта.
  • Используйте controls для удобства пользователя.
  • Для видео добавляйте субтитры через <track> .
Частая ошибка — отсутствие альтернативного текста для изображений и некорректные источники файлов.

Лучшие практики

  • Оптимизируйте изображения и видео для быстрой загрузки.
  • Проверяйте доступность мультимедийных элементов (alt, title, track).
  • Проверяйте кроссбраузерную совместимость медиаформатов.
  • Используйте современные форматы изображений (WebP, SVG) и видео (MP4, WebM).
  • По возможности используйте CDN для хранения больших файлов.
Соблюдение этих практик поможет сделать сайт быстрым, доступным и удобным для всех пользователей.
