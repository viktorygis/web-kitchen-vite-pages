Медиа-запросы в CSS

Медиа-запросы — это специальная возможность в CSS, которая позволяет сайту менять внешний вид в зависимости от устройства, на котором он открыт.

Например, на большом экране меню может быть сбоку, а на телефоне — сверху. Всё это делается с помощью медиа-запросов.

Media queries — основа адаптивного (responsive) дизайна. Благодаря им сайты выглядят удобно и красиво на любом экране: от телевизора 📺 до смартфона 📱.

Зачем нужны? Чтобы каждый пользователь видел сайт, который удобно читать и использовать на своём устройстве.

Попробуйте добавить медиа-запросы в свой проект и увидеть, как легко сайт может "примерить" новый стиль!

Синтаксис медиа-запросов

Медиа-запрос — это особое правило в CSS, которое говорит браузеру: "Применяй эти стили, только если выполнено определённое условие."
Обычно медиа-запрос начинается с @media , после него указывается условие (например, ширина экрана), а внутри — CSS-правила, которые применяются только при выполнении этого условия.
Так сайт подстраивается под разные устройства: большие мониторы, планшеты, телефоны.

Пример: меняем фон для маленьких экранов


@media (max-width: 600px) {
  body {
    background-color: lightblue; /* Голубой фон для маленьких экранов */
  }
}
На экранах ≤ 600px фон станет голубым
  • 1️⃣ @media — ключевое слово для медиа-запроса.
  • 2️⃣ (max-width: 600px) — условие: если ширина экрана ≤ 600px, стили внутри скобок применяются.
  • 3️⃣ CSS-правила — внутри скобок указываете, что изменять (цвет, размер, выравнивание и т.д.).
Можно комбинировать условия:
@media (max-width: 600px) and (orientation: portrait) { ... }
Что можно проверять:
  • Ширину: max-width , min-width
  • Высоту: max-height , min-height
  • Ориентацию: orientation: portrait или landscape
  • Разрешение экрана: resolution
Важно: Без условия (например, max-width ) медиа-запрос не работает!

Пример: адаптация заголовка на телефоне


@media (max-width: 600px) {
  h1 {
    font-size: 20px; /* Заголовок становится меньше */
    color: blue;     /* Цвет текста синий */
  }
}

Заголовок станет меньше и синим на экранах ≤ 600px

  • На экранах шире 600px — все стили остаются обычными.
  • На телефоне или маленьком экране — заголовок <h1> станет меньше и синим.
💡 Совет: Измените цвет фона, размер текста или расположение элементов для разных устройств — это поможет лучше понять, как работают медиа-запросы!

Медиа-запросы по ориентации экрана

Ориентация экрана — это положение устройства:
portrait — вертикально (например, телефон в обычном режиме) 📱
landscape — горизонтально (например, планшет или телефон, повернутый набок) 📟

Медиа-запросы позволяют изменять стили сайта в зависимости от ориентации экрана.
Это особенно полезно для приложений, игр, видео и любого контента, который должен выглядеть по-разному в разных положениях устройства.

Синтаксис: После слова orientation обязательно ставь двоеточие : и указывай значение portrait или landscape .
Всё это пишется внутри круглых скобок () — без них запрос не сработает!

Примеры: меняем фон в зависимости от ориентации


@media (orientation: portrait) {
  body {
    background-color: lightblue; /* Голубой фон для вертикальной ориентации */
  }
}

@media (orientation: landscape) {
  body {
    background-color: lightgreen; /* Зелёный фон для горизонтальной ориентации */
  }
}
Если устройство вертикально — фон голубой Если устройство горизонтально — фон зелёный

Зачем использовать?

  • Менять расположение элементов, кнопок, текста или изображений в зависимости от ориентации.
  • Улучшать удобство для игр, видео, презентаций, интерактивных приложений.
  • Делать сайт или приложение более адаптивным и приятным для пользователя.

Как объединять условия в медиа-запросах

Иногда нужно менять стили только если выполнено сразу несколько требований, например:
устройство узкое и находится в портретной ориентации.
Для этого используют ключевое слово and между условиями.
@media (max-width: 768px) and (orientation: portrait) {}

Зачем это нужно?

Такой подход позволяет создавать ещё более точную и гибкую адаптацию макета — например,
показывать специальные стили только для телефонов, которые пользователь держит вертикально.

Пример: комбинированный медиа-запрос


@media (max-width: 768px) and (orientation: portrait) {
  body {
    background-color: lightcoral; /* Красный фон для портретной ориентации на узких экранах */
  }
}
Если ширина ≤ 768px и экран вертикальный — фон станет красным
  • 1️⃣ (max-width: 768px) — ширина экрана меньше или равна 768px.
  • 2️⃣ and — объединяет условия: нужно проверить ещё одно.
  • 3️⃣ (orientation: portrait) — экран находится в вертикальном положении.
  • 4️⃣ Только если оба условия выполнены , стили внутри фигурных скобок {} будут применены.
💡 Совет: Чем больше условий в медиа-запросе, тем точнее можно управлять адаптивностью сайта. Комбинируйте параметры для продвинутых решений!

Часто используемые типы и параметры медиа-запросов

Медиа-запросы позволяют адаптировать сайт под разные устройства, возможности пользователя и режимы работы браузера.
Вот основные параметры, которые чаще всего применяют:

  • width, min-width, max-width — ширина области просмотра (экрана).
    Используется для создания брейкпоинтов: например, для мобильных, планшетов и компьютеров.
  • height, min-height, max-height — высота области просмотра.
    Редко, но полезно, если важна вертикальная прокрутка или высота окна.
  • orientation — ориентация экрана: portrait (вертикально) и landscape (горизонтально).
    Удобно для изменения макета на телефонах и планшетах при вращении устройства.
  • pointer, hover — есть ли мышь и поддержка наведения (hover).
    Позволяет отличать сенсорные устройства от десктопов и менять стили для кнопок и меню.
  • resolution — плотность пикселей (например, Retina-экраны).
    Позволяет подгружать изображения с высоким разрешением для современных дисплеев.
  • color-scheme — предпочтения пользователя по теме: светлая или тёмная.
    Сайт может автоматически переключаться между светлой и тёмной темой.
Практический совет: Самые популярные параметры — min-width и max-width .
Их используют для создания брейкпоинтов — точек, в которых макет "переключается" под размер экрана (например, мобильный, планшет, десктоп).

Пример брейкпоинта:


@media (max-width: 900px) {
  .menu {
    flex-direction: column;
  }
}

Примеры медиа-запросов

base
Адаптивная сетка
Меняет количество колонок на мобильных устройствах.

@media (max-width: 600px) {
  .grid {
    grid-template-columns: 1fr;
    gap: 12px;
  }
}
breakpoint
Два брейкпоинта
Сетка меняется на планшете и телефоне.

@media (max-width: 900px) {
  .grid {
    grid-template-columns: 1fr 1fr;
  }
}
@media (max-width: 600px) {
  .grid {
    grid-template-columns: 1fr;
  }
}
orientation
Разная верстка в портретном режиме
Меняет фон, если устройство повернули вертикально.

@media (orientation: portrait) {
  body {
    background: #e3f2fd;
  }
}
dark mode
Тёмная тема
Стили для темной темы на устройствах.

@media (prefers-color-scheme: dark) {
  body {
    background: #181b1f;
    color: #eee;
  }
}
retina
Retina/HiDPI дисплеи
Стили для экранов с высокой плотностью пикселей.

@media (min-resolution: 2dppx) {
  .logo {
    background-image: url(logo@2x.png);
  }
}
hover
Только для устройств с мышкой
Ховер-эффекты только на десктопах.

@media (hover: hover) and (pointer: fine) {
  .button:hover {
    background: #1976d2;
    color: #fff;
  }
}

Советы по использованию media queries

Медиа-запросы — мощный инструмент для адаптивной верстки. Несколько простых правил помогут сделать ваш сайт удобным и современным на любых устройствах!

  • Используйте подход mobile-first
    Сначала пишите базовые стили для мобильных устройств, а затем добавляйте медиа-запросы для планшетов и десктопов. Так проще контролировать адаптивность и избегать лишних переопределений.
  • Старайтесь использовать минимум брейкпоинтов
    Добавляйте точки переключения только там, где действительно “ломается” дизайн, чтобы код был проще и быстрее работал.
  • Группируйте похожие стили с помощью SASS или CSS Custom Properties
    Это делает код более структурированным и легко изменяемым.
  • Проверяйте макет на разных устройствах и в DevTools
    Используйте режим Responsive в браузере или тестируйте на реальных телефонах и планшетах. Не забывайте про горизонтальную и вертикальную ориентацию!
  • Для сложных проектов используйте container queries
    Container queries позволяют делать адаптивные компоненты, которые меняют вид не только в зависимости от размера экрана, но и от размера самого контейнера. Пока поддержка не во всех браузерах, но это современный тренд.
  • Пишите понятные условия и комментарии
    Если медиа-запрос сложный, добавьте комментарий, чтобы другим разработчикам было ясно, зачем он нужен.
  • Тестируйте на доступность
    Не отключайте масштабирование, используйте относительные единицы ( em , rem , % ) вместо фиксированных px . Так ваш сайт будет удобнее для всех пользователей.
Совет: Думайте о каждом пользователе! Адаптивность — это не только про размеры, но и про удобство и доступность для всех.
