Анимация в CSS: transition и animation

CSS-анимация позволяет создавать плавные переходы, эффекты наведения, а также сложные движения и циклические эффекты без JS.
Основные инструменты: transition для простых изменений и animation для сложных, многошаговых сценариев.

Свойство transition

  • Позволяет плавно анимировать изменение CSS-свойств (например, цвет, размер, тень, позицию).
  • Работает при смене состояния (hover, focus, active, через JS).
  • Базовый синтаксис: transition: свойство продолжительность функция-затухания задержка;
Чем конкретнее указано свойство (transition: background 0.4s), тем проще контролировать эффекты.
Не используйте transition: all без необходимости.
.box {
  transition: background 0.4s, transform 0.3s;
}

.box:hover {
  background: #1976d2;
  transform: scale(1.15);
}

Свойство animation, @keyframes и параметры

  • Позволяет описать сложную, многошаговую анимацию с помощью @keyframes.
  • Управляет длительностью, задержкой, количеством повторов, направлением, заполнением, запуском/паузой и даже временной шкалой.
  • Анимация может запускаться автоматически или по событию, без использования JS.
Используйте animation для повторяющихся, циклических и сложных движений (например, индикаторы загрузки, bounce, fade-in).
Сокращённая запись:
animation: bounce 0.8s ease 0.3s 3 alternate none running;

Основные свойства animation:

  • animation-name — имя анимации (соответствует @keyframes).
  • animation-duration — длительность одного цикла анимации (например, 1s, 500ms).
  • animation-timing-function — функция изменения скорости (например, linear, ease, ease-in, cubic-bezier).
  • animation-delay — задержка перед стартом анимации.
  • animation-iteration-count — сколько раз повторить анимацию (1, infinite, 3 и т.д.).
  • animation-direction — направление проигрывания (normal, reverse, alternate, alternate-reverse).
  • animation-fill-mode — поведение до/после анимации (none, forwards, backwards, both).
  • animation-play-state — запуск/пауза (running, paused).
  • animation-timeline — позволяет запускать анимацию по времени, скроллу и т.д. (поддержка ограничена).
  • animation-range-start / animation-range-end — задаёт диапазон времени или скролла для анимации (экспериментально).

Пример использования свойств:

@keyframes bounce {
  0%   { transform: translateY(0); }
  40%  { transform: translateY(-24px); }
  100% { transform: translateY(0); }
}

.loader {
  animation-name: bounce;
  animation-duration: 0.8s;
  animation-timing-function: ease;
  animation-delay: 0.3s;
  animation-iteration-count: 3;
  animation-direction: alternate;
  animation-fill-mode: none;
  animation-play-state: running;
  animation-timeline: auto;
  animation-range-start: normal;
}
Совет: Для большинства задач достаточно указать animation-name, animation-duration и animation-iteration-count. Остальные параметры — для сложных сценариев.

Примеры transition и animation

transition
Плавное наведение
Изменение цвета и масштаба с помощью transition.
Hover
.box {
  color: #1565c0;
  transition: all 0.6s cubic-bezier(.48,.3,.55,1.15);
}

.box:hover {
  background: #1976d2;
  transform: scale(1.17) rotate(-5deg);
  color: #fff;
}
animation
Bounce (keyframes)
Многошаговая анимация с помощью @keyframes и animation.
Bounce
@keyframes bounce {
  0%   { transform: translateY(0); }
  40%  { transform: translateY(-22px); }
  60%  { transform: translateY(-12px); }
  100% { transform: translateY(0); }
}

.demo {
  animation: bounce 1.3s infinite;
}
iteration/delay
Повтор и задержка
Анимация может стартовать с задержкой и выполняться определённое число раз.
Bounce
.loader {
  animation: bounce 0.8s 0.3s 3 alternate;
}

/* Значения:
   bounce — имя keyframes
   0.8s — длительность
   0.3s — задержка перед стартом
   3 — количество повторов
   alternate — меняет направление на каждом цикле
*/
fill-mode
animation-fill-mode
Управляет тем, как элемент выглядит до/после анимации.
Fade-in
.loader {
  animation: fade-in 1s forwards;
}

/* forwards — сохраняет конечное состояние после анимации */
@keyframes fade-in {
  from { opacity: 0; }
  to   { opacity: 1; }
}

Советы по анимации в CSS

  • Старайтесь анимировать только transform и opacity — это быстрее и плавнее всего.
  • Избегайте анимации размеров (width, height), тени и фона на больших блоках — это может замедлять интерфейс.
  • Для последовательных анимаций используйте animation-delay и animation-fill-mode.
  • Проверяйте, чтобы анимации не мешали восприятию контента и не вызывали дискомфорта.
  • Тестируйте на разных устройствах и браузерах.
  • Используйте prefers-reduced-motion для отключения анимаций по желанию пользователя: 
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}
Совет: Краткие и плавные анимации делают интерфейс современным и "живым", но не перебарщивайте.
