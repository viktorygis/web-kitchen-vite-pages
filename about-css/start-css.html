Основы CSS

CSS (Cascading Style Sheets) — это язык, используемый для описания внешнего вида и форматирования HTML-документов.
С помощью CSS можно управлять цветами, шрифтами, отступами, выравниванием, анимациями и другими аспектами оформления сайта.

Представьте, что HTML — это каркас здания 🏗️, а CSS — это дизайн интерьера, цвета стен, мебель и стиль.
С помощью CSS вы можете:

  • Изменять цвета 🎨, шрифты ✍️, размеры 📏
  • Управлять расположением элементов на странице
  • Добавлять эффекты, анимации и делать сайт визуально привлекательным

<p style="color: blue;">Привет, мир!</p>

Привет, мир!

Синтаксис CSS

CSS состоит из трёх главных частей:
1️⃣ Селектор — указывает, к какому элементу применять стили.
2️⃣ Свойство — описывает, что мы хотим изменить.
3️⃣ Значение — указывает, как именно мы хотим изменить свойство. 


p {
  color: red;
}

Этот текст станет красным

  • p — селектор, который выбирает все абзацы на странице
  • color — свойство, задающее цвет текста
  • red — значение, указывающее, что цвет текста должен быть красным

⚠️ Важно:
— Свойство и значение разделяются двоеточием :
— В конце каждой строки ставится точка с запятой ;

1. Подключение CSS

CSS можно добавить к HTML-документу 4 основными способами:

  • Встроенный стиль
  • Внутренний стиль
  • Внешний файл
  • Импорт CSS внутри CSS

Встроенный стиль (inline)

Добавляются прямо в атрибут style внутри тега HTML.

💡 Подходит для единичных изменений, но для больших проектов неудобен.

<h1 style="color: red;">Заголовок</h1>

💡 Плюсы:

  • Быстро и просто.

⚠️ Минусы:

  • Стили трудно редактировать.
  • Невозможно использовать один стиль для нескольких элементов.

🎨 Когда использовать?
Только для тестирования или для единичных стилей, которые не повторяются.

Внутренний стиль (internal)

Добавляются в блок <style> внутри <head> в HTML-документе.

💡 Удобно для небольших проектов, когда все стили находятся в одном месте.

<head>
  <style>
    body { background-color: lightblue; }
  </style>
</head>

💡 Плюсы:

  • Все стили находятся в одном месте внутри HTML-документа.

⚠️ Минусы:

  • Удобно только для небольших проектов.

🎨 Когда использовать?
Внутренние стили удобны для небольших страниц, где все стили нужно сосредоточить в одном месте. Но для больших проектов лучше использовать внешние стили. 🚀

Внешний файл (external)

Самый удобный и универсальный способ стилизовать сайты!

Все стили хранятся в отдельном файле с расширением .css , а подключаются к HTML через тег <link> . Такой подход облегчает поддержку и повторное использование стилей для разных страниц.

  1. Создай файл с расширением .css , например, styles.css .
  2. Добавь в него свои стили: 
    h1 {
  color: orange;
  font-size: 50px;
}
  3. Подключи этот файл в секции <head> твоего HTML-документа: 
    <head>
  <link rel="stylesheet" href="styles.css">
</head>
  • <link> сообщает браузеру, что нужно загрузить внешний файл стилей.
  • Атрибут rel="stylesheet" указывает, что это таблица стилей CSS.
  • Атрибут href="styles.css" задаёт путь к твоему CSS-файлу.

💡 Совет:
Если твой CSS-файл лежит в отдельной папке (например, css/style.css ), обязательно укажи путь:
<link rel="stylesheet" href="css/style.css">

Медиа-условия для <link>:

<link rel="stylesheet" href="print.css" media="print">
<link rel="stylesheet" href="wide.css" media="screen and (min-width: 768px)">

Атрибут media позволяет загружать/применять стили по условиям, не блокируя критический CSS.

Импорт файлов CSS внутри CSS ( @import )

Директива @import позволяет подключать один CSS-файл внутри другого.
Удобно для модульной структуры, но используйте осторожно из‑за влияния на производительность.

Синтаксис

/* Должно идти вверху файла, после @charset/@layer, до обычных правил */
@import url("reset.css");
@import "theme.css"; /* допустимы кавычки без url() */

body { font-family: Arial, sans-serif; }

Импорт по медиa-условиям

@import url("print.css") print;
@import url("wide.css") screen and (min-width: 768px);

Импорт с каскадными слоями

/* Современный приём: упорядочивание стилей слоями */
@import url("base.css") layer(base);
@import url("components.css") layer(components);

@layer utilities {
  /* локально объявленный слой */
  .sr-only { position: absolute; clip: rect(0,0,0,0); }
}

Порядок и каскад:

  • Правила из импортируемого файла вставляются на месте @import , раньше последующего кода.
  • @import должен располагаться в начале таблицы, после @charset и возможных объявлений @layer .
  • Относительные пути внутри импортируемого файла считаются относительно расположения этого файла.

⚠️ Производительность:
  • Каждый @import — это дополнительный HTTP‑запрос. Цепочки импортов могут замедлять отрисовку.
  • В продакшне предпочтительно подключать стили через <link> и объединять их сборщиком (Webpack, Vite, Gulp, Parcel).
  • Если нужен условный импорт — чаще эффективнее использовать media у <link> , чем @import внутри CSS.

Когда использовать что?

  • <link> — предпочтительно почти всегда (кэширование, контроль загрузки, media, preload).
  • @import — удобно для модульности в исходниках, если затем происходит сборка/объединение.

Типичные ошибки

  • Размещение @import после обычных CSS‑правил — браузер проигнорирует такую директиву.
  • Длинные цепочки @import (A импортирует B, B — C) — возможны задержки загрузки.
  • Путаница с препроцессорами: @use / @import в Sass — это не нативный CSS.

🌟 Рекомендуется использовать внешний файл через <link> для удобства, скорости и повторного использования стилей на разных страницах.
Директива @import уместна на этапе разработки/сборки или для небольших, не критичных по скорости проектов.

2. Селекторы CSS

Селектор — это то, что указывает, к какому элементу HTML мы хотим применить стили.

🗳️ Существует три основных типа селекторов:

  1. Селектор по тегу : выбирает все элементы одного типа (например, все <p> или <h1> ).
  2. Селектор по классу : выбирает элементы с определённым классом.
  3. Селектор по ID : выбирает уникальный элемент с заданным ID. 
/* Селектор по тегу */
p {
  color: blue;
}

/* Селектор по классу */
.my-class {
  font-size: 20px;
}

/* Селектор по ID */
#unique-id {
  text-align: center;
}

Как выбрать элементы на странице?
● Теги — для стилизации всех элементов одного типа.
● Классы — для стилизации группы элементов.
● ID — для уникальных элементов, которые заслуживают особого внимания.

Селектор по тегу

Селектор по тегу — выбирает все элементы определённого типа: 

p { color: green; }

🔍 Селектор по тегу — это самый простой и базовый способ выбрать элементы для стилизации. Он применяет стиль ко всем элементам одного типа на странице.

Например, если ты хочешь, чтобы все абзацы <p> на твоей странице были зелёными, просто используй селектор по тегу. 

<style>
  p {
    color: green; /* Зелёный цвет текста */
    font-size: 18px; /* Размер текста 18px */
  }
</style>

⚠️ Будь осторожен! Если у тебя много разных <p> на странице, изменения затронут все такие элементы. В таких случаях лучше использовать классы или ID.

Пример: 
<a href="#">Первая ссылка</a>
<a href="#">Вторая ссылка</a>
<a href="#">Третья ссылка</a>

<style>
  a {
    color: red;
  }
</style>

Селектор по классу

Селектор по классу — выбирает элементы с определённым классом (начинается с точки): 

.my-class { font-size: 20px; }

🔍 Селектор по классу — это как персональный стилист для группы элементов. Он позволяет применить один и тот же стиль ко многим элементам, даже если они разных типов.

Всё, что нужно — добавить к элементу атрибут class , а потом в CSS написать имя этого класса с точкой . перед ним.

1️⃣ Добавь класс к элементу в HTML: 
<p class="highlight">Я важный текст!</p>
<p class="highlight">И я тоже важный!</p>
2️⃣ Напиши стиль для класса в CSS: 
.highlight {
  color: green; /* Цвет текста */
  font-weight: bold; /* Жирный текст */
}

⚡ Когда использовать селекторы по классу?

  • Когда нужно применить стиль к определённой группе элементов.
  • Это удобнее, чем селекторы по тегу, если на странице есть элементы одного типа, но они должны выглядеть по-разному.

💡 Совет: Используй понятные имена для классов (например, highlight, button-primary, text-danger).

Пример: 
<h1 class="shrek">Это важный заголовок</h1>
<p class="shrek">А это важный абзац</p>
<p>Обычный текст без стиля</p>

<style>
  .shrek {
    color: green; /* Цвет текста */
    font-weight: bold; /* Жирный текст */
  }
</style>

Селектор по идентификатору

Селектор по идентификатору — выбирает элемент с определённым ID (начинается с решётки): 

#my-id { border: 1px solid black; }

🔍 Селектор по ID — это как VIP-пропуск для элемента. Он используется для стилизации только одного уникального элемента на странице.

Каждый ID должен быть уникальным, поэтому этот метод отлично подходит для выделения чего-то особенного.

1️⃣ Добавь атрибут id к элементу в HTML: 
<h1 id="main-title">Я особенный заголовок</h1>
2️⃣ Напиши стиль для этого ID в CSS, добавив перед именем ID символ решётки # : 
#main-title {
  color: orange; /* Оранжевый текст */
  text-align: center; /* Центрирование текста */
}

⚡ Когда использовать селекторы по ID?

  • Когда нужно выделить один уникальный элемент.
  • Например, для заголовка страницы, главной кнопки или особенного абзаца.

⚠️ Важно: Один ID может использоваться только для одного элемента на странице! Если нужно применить стиль к нескольким элементам, используй классы вместо ID.

Углублённые селекторы

В CSS есть продвинутые селекторы, которые позволяют точно выбирать элементы в зависимости от их расположения в структуре HTML.

Селектор потомков (descendant)

Селектор потомков выбирает все элементы, которые находятся внутри другого элемента, независимо от уровня вложенности .
Это как сказать: "Эй, выбери всё, что находится в этом доме, независимо от комнаты!" 🏠 

div p {
  color: blue; /* Все <p>, находящиеся внутри <div>, станут синими */
}

Пример:

<div>
	  <p>Первый абзац</p>
	  <section>
	    <p>Вложенный абзац</p>
	  </section>
	</div>

	<style>
	   div p {
	     color: blue;
	   }
	</style>

Первый абзац

Вложенный абзац

Оба <p> (и внутри <div> , и внутри <section> в <div> ) получат стиль, потому что они потомки <div> .

Ещё пример:

<section>
  <p>Абзац внутри первого раздела</p>
  <p>Ещё один абзац в первом разделе</p>
</section>

<section>
  <p>Абзац внутри второго раздела</p>
</section>

<p>Абзац, который не должен стать зелёным</p>

<style>
  section p {
    color: green;
  }
</style>

Абзац внутри первого раздела

Ещё один абзац в первом разделе

Абзац внутри второго раздела

Абзац, который не должен стать зелёным

Здесь зелёными станут только те <p> , которые находятся внутри <section> .

Селектор дочерних элементов (child)

Селектор дочерних элементов выбирает только те элементы, которые являются прямыми потомками родителя.
Используется символ > между родителем и дочерним элементом. 

div > p {
  font-size: 18px; /* Только прямые дети <div> */
}

Пример:

<div>
  <p>Прямой потомок (получит стиль)</p>
  <section>
    <p>Вложенный потомок (не получит стиль)</p>
  </section>
</div>

Прямой потомок (получит стиль)

Вложенный потомок (не получит стиль)

Здесь стиль применится только к первому <p> — он прямой потомок <div> . Вложенный <p> не затронется.

Соседний селектор (adjacent sibling)

Соседний селектор + позволяет выбрать элемент, который идёт сразу после другого .
Например, ты хочешь выделить только первый пункт после заголовка — это твой инструмент! 😎 

h1 + p {
  color: orange; /* Сделать текст оранжевым */
  font-size: 20px; /* Увеличить размер шрифта */
}
HTML-пример: 
<h1>Это заголовок</h1>
<p>Этот абзац идёт сразу после заголовка и получит стиль</p>
<p>А я — обычный абзац, стиль не для меня</p>

Сравнение: какой способ стилизации выбрать?

Селектор Когда использовать Пример
По тегу Для стилизации всех элементов одного типа p { color: green; }
По классу Для группы элементов (даже разных типов) .note { font-weight: bold; }
По ID Для одного уникального элемента #main-title { text-align: center; }
Потомок Для всех вложенных элементов div p { ... }
Прямой потомок Только для прямых детей ul > li { ... }
Соседний Только для элемента сразу после другого h1 + p { ... }
