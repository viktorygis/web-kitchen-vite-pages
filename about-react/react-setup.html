Настройка React-проекта

В этой инструкции собраны практические шаги, которые делают разработку React удобной: настройка редактора, форматирование, линтинг, TypeScript, инструменты разработчика и понимание того, как работает компиляция JSX.

Зачем это нужно

  • Меньше ошибок — линтер и TypeScript ловят проблемы раньше, чем браузер.
  • Единый стиль кода — Prettier форматирует код одинаково у всей команды.
  • Быстрее дебаг — React DevTools помогает понимать, почему компонент ререндерится.
  • Понимание сборки — вы знаете, что именно “компилируется”, и где искать проблемы.

Рекомендуется: начать с Vite + React, затем добавить Prettier, ESLint и при необходимости TypeScript.

Настройка редактора VS Code

Базовые расширения

  • ESLint — подсветка проблем и автофикс (расширение от Microsoft).
  • Prettier - Code formatter — автоформатирование кода.
  • TypeScript and JavaScript Language Features — встроено, но важно держать VS Code обновлённым.
  • React Developer Tools — это расширение браузера (Chrome/Firefox), не VS Code, но обязательно.
  • EditorConfig for VS Code (опционально) — если команда использует .editorconfig.

Полезные настройки VS Code (settings.json)

Эти настройки включают автоформатирование при сохранении и позволяют VS Code “доверять” Prettier/ESLint. 

{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",

  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },

  "eslint.validate": ["javascript", "javascriptreact", "typescript", "typescriptreact"],

  "files.eol": "\n",
  "files.insertFinalNewline": true,
  "files.trimTrailingWhitespace": true
}

Если форматирование “не срабатывает”, проверьте: установлен ли Prettier, выбран ли он как форматтер по умолчанию, и нет ли конфликта с другими форматтерами.

Форматирование кода (Prettier)

Prettier отвечает только за стиль (отступы, кавычки, переносы строк). Он не “проверяет правильность логики” — это задача линтера и TypeScript.

Установка

npm i -D prettier

Пример .prettierrc

{
  "semi": true,
  "singleQuote": true,
  "printWidth": 100,
  "trailingComma": "all"
}

Игнорирование файлов (.prettierignore)

dist
build
node_modules
coverage

Обычно Prettier запускают автоматически “на сохранение” (VS Code) и дополнительно — в CI (опционально).

Линтеры кода (ESLint)

ESLint помогает находить ошибки и антипаттерны: неиспользуемые переменные, неправильные зависимости хуков, потенциальные баги в коде и т.д.

Если вы используете Vite

В Vite ESLint обычно настраивается явно (в отличие от CRA, где многое “из коробки”).

Минимальная установка (пример)

npm i -D eslint eslint-plugin-react eslint-plugin-react-hooks eslint-plugin-import

Пример конфигурации eslint (очень упрощённо)

// eslint.config.js или .eslintrc.cjs (вариант зависит от выбранного подхода)
module.exports = {
  env: { browser: true, es2022: true, node: true },
  extends: [
    "eslint:recommended",
    "plugin:react/recommended",
    "plugin:react-hooks/recommended"
  ],
  settings: {
    react: { version: "detect" }
  }
};

Важно: для React используйте правила react-hooks — они предотвращают критические ошибки с хуками.

Использование TypeScript в React

TypeScript добавляет типы в JavaScript и помогает: автодополнением, безопасным рефакторингом и ранним обнаружением ошибок.

Когда TypeScript особенно полезен

  • сложные формы и валидации;
  • много данных из API и преобразований;
  • большая команда и долгий срок жизни проекта.

Как начать (Vite + React + TS)

npm create vite@latest my-app -- --template react-ts

Если проект уже существует на JS

  • можно добавлять TypeScript постепенно — сначала типизировать новые файлы;
  • переименовывать компоненты с .jsx на .tsx по мере необходимости;
  • начать с типизации данных: ответы теста, пользовательские данные, результат API.

Пример простых типов (props + данные)

type UserData = {
  name: string;
  email: string;
};

type SubmitPayload = {
  user: UserData;
  answers: Record<string, number>;
  patterns: string[];
  date: string;
};

Подход “постепенно” обычно лучше, чем попытка типизировать весь проект за один день.

Инструменты разработчика React (React Developer Tools)

React Developer Tools — расширение для Chrome/Firefox, которое добавляет вкладки Components и Profiler в DevTools браузера.

Что полезно смотреть

  • Props/State — какие данные получает компонент и что у него в состоянии.
  • Компонентное дерево — быстро понять структуру страницы.
  • Profiler — измерение производительности и причин ререндеров.

Типичный кейс: “почему компонент ререндерится?”

  • Откройте вкладку Profiler.
  • Нажмите запись, сделайте действие в UI, остановите запись.
  • Посмотрите, какие компоненты ререндерятся и сколько времени это занимает.

React DevTools — один из самых практичных инструментов для ежедневной разработки.

Компиляция React: JSX, Babel/SWC и React Compiler

1) Что “компилируется” в React

Браузер не понимает JSX напрямую. Код вида <div>Hello</div> должен быть преобразован в JavaScript. Это делает сборщик (например Vite) через транспайлер (Babel или SWC).

Пример: JSX → JS (упрощённо)

// JSX
const el = <button onClick={onClick}>OK</button>;

// После трансформации (примерно)
const el = React.createElement("button", { onClick }, "OK");

2) Babel и SWC — что выбрать

  • Babel — самый распространённый транспайлер, огромная экосистема плагинов.
  • SWC — очень быстрый транспайлер на Rust, часто даёт ускорение сборки.

В большинстве случаев достаточно “дефолтов” Vite/плагина React — не нужно вручную настраивать Babel, пока нет специфических требований.

3) React Compiler (опционально)

React Compiler — это отдельная идея: автоматическая оптимизация ререндеров (вместо ручного useMemo/useCallback везде). Это не “обязательная часть” React-приложения и внедряется постепенно.

  • Если у вас небольшое/среднее приложение — обычно можно жить без него.
  • Если есть сложные ререндеры и производительность — можно изучать и пробовать точечно.

Чеклист настройки (коротко)

  1. Установить расширения VS Code: ESLint, Prettier.
  2. Включить formatOnSave и автофикс ESLint по сохранению.
  3. Добавить .prettierrc и .prettierignore.
  4. Настроить ESLint (минимально: react + react-hooks).
  5. При необходимости добавить TypeScript постепенно.
  6. Установить React Developer Tools в браузер.
  7. Понимать базовый пайплайн: JSX компилируется (Babel/SWC), затем бандлится (Vite/Rollup).
