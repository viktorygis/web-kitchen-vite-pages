Работа с данными и data() в React

Данные — основа динамики в React.
Вы учитесь получать, хранить, изменять и передавать данные между компонентами.
State и props — инструменты для управления состоянием.
Поток данных всегда идет сверху вниз.
На этой странице — всё, чтобы уверенно работать с данными в React!

Отличие данных и состояния (state) в React от Vue:

  • В React нет реактивности как в Vue — изменения state вызывают перерисовку.
  • State в React может быть любым типом (объекты, массивы, примитивы).
  • Данные передаются только сверху вниз через props.
  • Для сложной логики состояния часто используют дополнительные библиотеки (Redux, MobX и др.).

Понятия о данных в React

  • Данные (data) — любая информация, которую отображает или использует компонент. Например, текст, изображения, списки и т. д.
  • State — внутренние данные компонента, которые могут меняться. Используется для хранения локального состояния.
  • Props — данные, которые компонент получает от родителя. Они неизменяемы внутри компонента.
  • Поток данных — передача данных сверху вниз (от родителя к детям). Компоненты не могут изменять props.
  • Списки данных — массивы объектов для отображения коллекций. Каждый элемент должен иметь уникальный ключ (key).
  • Загрузка данных — получение данных из внешних источников (API) с помощью fetch и useEffect.
  • Хук — специальная функция, которая позволяет использовать состояние и другие возможности React в функциональных компонентах.

Поток данных в React

Данные в React передаются только сверху вниз — от родителя к дочерним компонентам через props .
Изменять данные можно только там, где они хранятся (в state родителя).

Схема потока данных


App (state)
 ├─→ List (props)
 │     ├─→ ListItem (props)
Важно: Компоненты не могут изменять props — только читать и отображать.
Для изменения данных используйте функции обновления state в родительском компоненте.
Это нужно для предсказуемости и стабильности приложения!

Пример потока данных от родителя к дочернему компоненту

function App() {
  //Родительский компонент с состоянием
  const [items, setItems] = React.useState([]);

  //Передача массива в дочерний компонент через props
  return (
    <div>
      <!-- Компонент List получает items как props -->
      <List items={items} />
    </div>
  );
}

//Дочерний компонент List получает props и отображает список
function List({ items }) {
  //items - массив из props
  return (
    <ul>
      {items.map(item => (
        <!-- Передача каждого item в ListItem -->
        <ListItem key={item.id} item={item} />
      ))}
    </ul>
  );
}

//Еще один дочерний компонент, получает один item через props
function ListItem({ item }) {
  //item - объект из props
  return <li>{item.name}</li>;
}

Объяснение кода

  • App — родительский компонент с состоянием (state) items.
  • App передает items в List через props.
  • List получает props и отображает список, передавая каждый item в ListItem.
  • ListItem получает один item через props и отображает его.
  • Поток данных идет только сверху вниз: App → List → ListItem.
  • Компоненты не могут изменять props, только читать и отображать.

Загрузка данных (fetching)

Для загрузки данных из API используйте fetch() и хук useEffect .

fetch() — это встроенный метод для выполнения HTTP-запросов. У него есть два основных параметра: URL и объект настроек.
HTTP-запрос это запрос, отправляемый клиентом на сервер для получения или отправки данных. Это может быть GET, POST, PUT, DELETE и другие методы.
useEffect — хук, который позволяет выполнять побочные эффекты в функциональных компонентах. Например, загрузку данных, подписки или изменение DOM.
Хук принимает функцию и массив зависимостей. Если массив пустой, эффект выполняется только при монтировании компонента.

Пример загрузки данных

//Компонент загружает и отображает список пользователей
function Users() {
  //State для хранения пользователей
  const [users, setUsers] = React.useState([]);

  //useEffect для загрузки данных при монтировании
  React.useEffect(() => {
    //fetch для получения данных с API
    fetch("https://jsonplaceholder.typicode.com/users")
      //Преобразование ответа в JSON
      .then(res => res.json())
      //Обновление state с полученными данными
      .then(data => setUsers(data));
  }, []);

  //Рендер списка пользователей из state
  return (
    <ul>
      <!-- Отображение каждого пользователя -->
      {users.map(u => <li key={u.id}>{u.name}</li>)}
    </ul>
  );
}

Что происходит в коде:

  • Создается state для хранения пользователей.
  • useEffect загружает данные при монтировании.
  • fetch получает данные с API и преобразует в JSON.
  • setUsers обновляет state с полученными данными.
  • Компонент рендерит список пользователей из state.

Данные в state

State — это внутренние данные компонента, которые могут меняться. Используется для хранения локального состояния.
В React state создается с помощью хука useState .
State может быть любым типом: объекты, массивы, примитивы (строки, числа, булевы).
Например, для хранения списка продуктов используйте массив объектов.

Пример использования данных в state

  • Списки продуктов, задачи, сообщения и т. д.
  • Формы с несколькими полями.
  • Настройки пользователя.
  • Любые данные, которые меняются со временем.

Пример данных в state


function Products() {
  //State для хранения массива продуктов, products - текущее состояние, setProducts - функция для обновления
  const [products, setProducts] = React.useState([
    //Начальное значение - массив продуктов
    { id: 1, name: "Ноутбук" },
    { id: 2, name: "Мышь" }
  ]);

  //Рендер списка продуктов из state
  return (
    //Отображение каждого продукта
    <ul>
      <!--Перебор массива products и создание li для каждого продукта -->
      {products.map(p => <li key={p.id}>{p.name}</li>)}
    </ul>
  );
}

Объяснение кода:

  • function Products() - это функциональный компонент.
  • useState - хук для создания state для компонента Products . Принимает начальное значение (массив продуктов с id и name ) и возвращает массив с текущим состоянием и функцией для его обновления. Где products - текущее состояние, setProducts - функция для обновления.
  • Внутри компонента возвращается JSX-разметка, которая отображает список продуктов.
  • Используется метод map() для перебора массива products и создания элемента li для каждого продукта.
  • Каждый элемент списка должен иметь уникальный ключ (key) - в данном случае используется id продукта.
  • Компонент рендерит список продуктов из state.

Передача данных через props

Props — это данные, которые компонент получает от родителя. Они неизменяемы внутри компонента.
Props позволяют передавать информацию и конфигурацию в дочерние компоненты.
Например, для передачи списка элементов используйте массив объектов.

Когда использовать props 👉

  • Передача данных от родителя к дочернему компоненту.
  • Конфигурация компонента (например, стили, классы и т. д.).
  • Передача функций-обработчиков событий.
  • Передача любых данных, которые не меняются внутри компонента.

Когда не использовать props ❌

  • Для хранения локального состояния компонента (используйте state).
  • Для передачи данных между несвязанными компонентами (используйте контекст или глобальное состояние).
  • Для изменения данных внутри компонента (props неизменяемы).

Пример передачи массива через props 👍

//Компонент List получает массив items через props и отображает список
function List(props) {
  return (
    <ul>
      <!-- Перебор массива items из props и создание li для каждого элемента -->
      {props.items.map(item => <li key={item.id}>{item.name}</li>)}
    </ul>
  );
}

// Использование	компонента List с передачей массива через props
<List items={[{id: 1, name: "Яблоко"}, {id: 2, name: "Груша"}]} />
Объяснение кода 📋
  • function List(props) - это функциональный компонент, который принимает props в качестве аргумента. Props - это объект, содержащий все переданные свойства.
  • Внутри компонента возвращается JSX-разметка, которая отображает список элементов.
  • Используется метод map() для перебора массива items из props и создания элемента li для каждого элемента.
  • Каждый элемент списка должен иметь уникальный ключ (key) - в данном случае используется id элемента.
  • Компонент List используется с передачей массива объектов через props .
  • Массив содержит объекты с id и name, которые отображаются в списке.
  • Компонент рендерит список элементов, полученных через props.

Обновление данных

Изменение данных в state с помощью функции обновления, возвращаемой useState .
В React state должен оставаться неизменяемым — для обновления создавайте новые копии объектов/массивов.
Например, для добавления элемента в массив используйте spread-оператор [...array, новый] .

Для чего это нужно?

  • Обновление интерфейса при изменении данных.
  • Реакция на действия пользователя (клики, ввод и т. д.).
  • Синхронизация с внешними источниками (API, WebSocket и т. д.).
  • Управление состоянием приложения.

Пример добавления элемента

//Компонент AddProduct для добавления продуктов в список
function AddProduct() {
  //State для хранения массива продуктов
  const [products, setProducts] = React.useState([]);
  //State для хранения значения input
  const [input, setInput] = React.useState("");

  //Функция для добавления нового продукта
  const handleAdd = () => {
   //Проверка на пустой ввод
    setProducts([...products, { id: Date.now(), name: input }]);
    setInput("");
  };

  //Рендер формы и списка продуктов
  return (
    <div>
      <!-- Поле ввода для названия продукта -->
      <input value={input} onChange={e => setInput(e.target.value)} placeholder="Название"/>
      <!-- Кнопка для добавления продукта -->
      <button onClick={handleAdd}>Добавить</button>
      <!-- Список продуктов -->
      <ul>
        {products.map(p => <li key={p.id}>{p.name}</li>)}
      </ul>
    </div>
  );
}
Объяснение кода 📋
  • Создается state для хранения массива продуктов и значения input.
  • Функция handleAdd добавляет новый продукт в массив, создавая новую копию с помощью spread-оператора.
  • После добавления input очищается.
  • Компонент рендерит форму для ввода и кнопку для добавления продукта.
  • Также отображается список продуктов из state.
  • Каждый элемент списка имеет уникальный ключ (key) - id продукта.
  • Компонент позволяет динамически добавлять продукты в список, обновляя state правильно.

Списки данных и ключи

Списки данных - это массивы объектов для отображения коллекций. Каждый элемент должен иметь уникальный ключ (key) для оптимальной работы React.
Key - это специальный атрибут, который помогает React идентифицировать элементы при обновлении списка. Он должен быть уникальным среди соседних элементов.
Используйте уникальные идентификаторы из данных (например, id) в качестве ключей. Если уникальных идентификаторов нет, можно использовать индекс массива, но это не рекомендуется для динамических списков.
Правильное использование ключей улучшает производительность и предотвращает баги при обновлении списков.

Когда использовать ключи?

  • При рендеринге списков с помощью map() или других методов перебора.
  • Когда элементы списка могут добавляться, удаляться или изменяться.
  • Для оптимизации производительности при обновлении DOM.

Для чего это нужно?

  • Для правильной идентификации элементов при обновлении списка.
  • Для предотвращения ошибок и багов при изменении списка.
  • Для оптимизации производительности при рендеринге списков.

Пример списка

//Компонент CityList получает массив городов через props и отображает список
function CityList(props) {
  return (
    //Отображение списка городов
    <ul>
      <!-- Перебор массива cities из props и создание li для каждого города -->
      {props.cities.map(city => <li key={city.id}>{city.name}</li>)}
    </ul>
  );
}
//Использование компонента CityList с передачей массива городов через props
<CityList cities={[{id: 1, name: "Москва"}, {id: 2, name: "Питер"}]} />
Объяснение кода 📋
  • Создается функциональный компонент CityList, который принимает props.
  • Внутри компонента возвращается JSX-разметка, которая отображает список городов.
  • Используется метод map() для перебора массива cities из props и создания элемента li для каждого города.
  • Каждый элемент списка имеет уникальный ключ (key) - в данном случае используется id города.
  • Компонент CityList используется с передачей массива объектов через props.
  • Массив содержит объекты с id и name, которые отображаются в списке.

Практика: мини-приложение "Список задач"

Создайте простое приложение для управления списком задач. Пользователь должен иметь возможность добавлять новые задачи, отмечать их как выполненные и видеть список всех задач.
Используйте state для хранения задач и props для передачи данных между компонентами.
Это упражнение поможет закрепить знания о работе с данными в React! 

<div id="root"></div>
<script type="text/babel">
function TodoApp() {
  const [tasks, setTasks] = React.useState([
    { id: 1, text: "Изучить React", done: false },
    { id: 2, text: "Сделать домашку", done: false }
  ]);
  const [input, setInput] = React.useState("");

  const addTask = () => {
    if (input.trim()) {
      setTasks([...tasks, { id: Date.now(), text: input, done: false }]);
      setInput("");
    }
  };

  const toggleTask = (id) => {
    setTasks(tasks.map(t => t.id === id ? { ...t, done: !t.done } : t));
  };

  return (
    <div>
      <input value={input} onChange={e => setInput(e.target.value)} placeholder="Новая задача"/>
      <button onClick={addTask}>Добавить</button>
      <ul>
        {tasks.map(task =>
          <li key={task.id} style={{ textDecoration: task.done ? "line-through" : "" }} onClick={() => toggleTask(task.id)}>
            {task.text}
          </li>
        )}
      </ul>
    </div>
  );
}
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<TodoApp />);
</script>
Объяснение кода 📋
  • Создается функциональный компонент TodoApp.
  • Используется useState для хранения массива задач и значения input.
  • Функция addTask добавляет новую задачу в массив, создавая новую копию с помощью spread-оператора.
  • Функция toggleTask переключает состояние задачи (выполнена/не выполнена) по id.
  • Компонент рендерит форму для ввода новой задачи и кнопку для добавления.
  • Также отображается список задач из state, каждая задача кликабельна для отметки выполнения.
  • Каждый элемент списка имеет уникальный ключ (key) - id задачи.
  • Компонент позволяет динамически управлять списком задач, обновляя state правильно.
В этом примере: хранение массива, добавление, обновление и отображение данных с помощью state!

Пример 2. Получение данных через fetch

В этом примере мы загружаем данные с внешнего API и отображаем их в списке.


function ApiData() {
  const [data, setData] = React.useState([]);
  React.useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/posts?_limit=3")
      .then(res => res.json())
      .then(setData);
  }, []);
  return (
    <ul>
      {data.map(post => <li key={post.id}>{post.title}</li>)}
    </ul>
  );
}
const root = ReactDOM.createRoot(document.getElementById('api-demo'));
root.render(<ApiData />);
Пояснение: useEffect для загрузки, setData для хранения.
Что происходит:
  • Создается компонент ApiData , который использует хук useState для хранения загруженных данных.
  • Хук useEffect используется для выполнения побочного эффекта - загрузки данных с внешнего API при монтировании компонента.
  • Метод fetch() отправляет HTTP-запрос к API и получает ответ.
  • Ответ преобразуется в JSON с помощью res.json() и передается в функцию setData для обновления состояния.
  • Компонент рендерит список загруженных данных, используя метод map() для создания элемента li для каждого элемента данных.
  • Каждый элемент списка имеет уникальный ключ (key) - в данном случае используется id элемента данных.
  • Компонент рендерится в элемент с id api-demo , отображая загруженные данные на странице.

Пример 3. Добавление элемента в массив

В этом примере мы добавляем элементы в массив, хранящийся в state, с помощью формы.


function AddDemo() {
  const [items, setItems] = React.useState([]);
  const [value, setValue] = React.useState("");
  const addItem = () => {
    if (value.trim()) setItems([...items, value]);
    setValue("");
  };
  return (
    <div>
      <input value={value} onChange={e => setValue(e.target.value)} />
      <button onClick={addItem}>Добавить</button>
      <ul>{items.map((item, i) => <li key={i}>{item}</li>)}</ul>
    </div>
  );
}
const root = ReactDOM.createRoot(document.getElementById('add-demo'));
root.render(<AddDemo />);
Пояснение: Обновление массива через spread-оператор, state не мутируется.
Что происходит:
  • Создается компонент AddDemo , который использует хук useState для хранения массива элементов и значения input.
  • Функция addItem добавляет новый элемент в массив, создавая новую копию с помощью spread-оператора.
  • После добавления input очищается.
  • Компонент рендерит форму для ввода нового элемента и кнопку для добавления.
  • Также отображается список элементов из state, каждый элемент кликабельна для отметки выполнения.
  • Каждый элемент списка имеет уникальный ключ (key) - в данном случае используется индекс элемента.
  • Компонент позволяет динамически добавлять элементы в список, обновляя state правильно.
