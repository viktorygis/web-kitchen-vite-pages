Введение Что такое контролируемый компонент? Двустороннее связывание в React Практика Советы Лучшие практики Примеры

Контролируемые компоненты и двухстороннее связывание в React

Компоненты — это функции, возвращающие JSX. Они делятся на контролируемые и неконтролируемые.

JSX — это расширение синтаксиса JavaScript, позволяющее писать HTML-подобный код внутри JS. JSX расшифровывается как JavaScript XML . XML — язык разметки, похожий на HTML. XML расшифровывается как eXtensible Markup Language (расширяемый язык разметки).

Контролируемые компоненты — это элементы формы, значения которых управляются React через state.
Они позволяют синхронизировать интерфейс и данные, обеспечивая предсказуемое поведение.

Неконтролируемые компоненты — это элементы формы, значения которых хранятся в DOM и читаются через ref.

Элементы формы — это input, select, textarea и т.д., с которыми пользователь взаимодействует для ввода данных.

ref — это способ получить доступ к DOM-элементу напрямую, минуя state. Обычно используется для неконтролируемых компонентов. Однако в большинстве случаев предпочтительнее использовать контролируемые компоненты. ref расшифровывается как "reference" (ссылка).

Двухстороннее связывание это когда изменение в UI обновляет данные, а изменение данных обновляет UI. В React это реализуется через state и обработчики событий.

UI это визуальное представление данных, с которым взаимодействует пользователь. UI расшифровывается как User Interface , что переводится как "пользовательский интерфейс".

state это источник правды для значения, а onChange обновляет state при вводе пользователя.
Так React синхронизирует интерфейс и данные, обеспечивая предсказуемое поведение.

В React нет v-model как во Vue!
Но двустороннее связывание делается через state и onChange .
Управляйте значением инпута через state — это и есть контролируемый компонент!

Что такое контролируемый компонент?

  • Контролируемый компонент — это элемент формы (input, select, textarea и т.д.), значение которого полностью управляется через state React-компонента.
  • Вся актуальная информация о значении хранится только в state, и любое изменение происходит через функцию onChange , которая обновляет state.
  • Такой подход делает React единственным источником правды (single source of truth) для данных формы: вы всегда знаете, какое значение сейчас у поля.
  • Преимущества: легко валидировать данные, реагировать на ввод в реальном времени, сбрасывать и модифицировать значения, реализовывать маски ввода и автосохранение.
  • Главная особенность: значение поля задается через prop value (или checked для checkbox/radio), а не само по себе.
  • Неконтролируемые компоненты — это такие, где значение хранится внутри DOM-элемента и читается только по запросу (например, через ref ).
  • Контролируемые компоненты — стандартный паттерн для всех современных форм в React.
Контролируемые компоненты позволяют синхронизировать интерфейс с логикой приложения, делают поведение формы предсказуемым и удобным для масштабирования.

Контролируемый компонент используется только в тех случаях, когда необходимо управлять состоянием формы из родительского компонента. Например, если нужно валидировать ввод, изменять значение на лету или сохранять его в глобальном состоянии.

Если же требуется просто получить значение поля при отправке формы, то можно использовать неконтролируемый компонент с ref.

Как реализовать двухстороннее связывание в React?

Зачем нужно двухстороннее связывание?

  • Чтобы интерфейс всегда отражал актуальные данные.
  • Чтобы данные обновлялись при вводе пользователя.
  • Чтобы легко валидировать и обрабатывать ввод в реальном времени.

Примеры двустороннего связывания:

  • Форма регистрации, где вводимые данные сразу проверяются на валидность.
  • Поисковая строка, которая фильтрует результаты по мере ввода.
  • Настройки профиля, которые сохраняются автоматически при изменении.

Как реализовать двустороннее связывание в React?

  • Используйте useState для хранения значения поля в state .
  • Устанавливайте value элемента формы из state .
  • Добавляйте onChange , который обновляет state при вводе пользователя.

value - это управляемое состояние инпута. Расшифровывается как "значение", и оно всегда соответствует текущему значению инпута.

onChange - это обработчик события, который вызывается при изменении значения инпута. Расшифровывается как "при изменении" и используется для обновления состояния.

e => setValue(e.target.value) - это функция-обработчик, которая получает событие изменения (e) и обновляет состояние (setValue) новым значением из инпута (e.target.value) . Это обеспечивает синхронизацию между вводом пользователя и состоянием компонента. 


function InputDemo() {
  const [value, setValue] = React.useState("");
  return (
    <input value={value} onChange={e => setValue(e.target.value)} />
  );
}
Это и есть "двустороннее связывание по-React": значение управляется через state и меняется через onChange.

Практика: контролируемый инпут

Описание: Создайте компонент, который будет отображать текстовое поле и параграф с текущим значением этого поля.

Что происходит в коде:

  • Используется useState для создания state -переменной text и функции setText для её обновления.
  • Элемент input получает значение из state (value={text}) и обновляет его при вводе (onChange={e => setText(e.target.value)}).
  • Параграф отображает текущее значение text , синхронизированное с инпутом. 

function ControlledDemo() {
  const [text, setText] = React.useState("");
  return (
    <div>
      <input value={text} onChange={e => setText(e.target.value)} placeholder="Введите текст"/>
      <p>Текущее значение: {text}</p>
    </div>
  );
}

Советы и типичные ошибки

  • ✔ Не забывайте указывать value для input — иначе он неконтролируемый!
  • ✔ onChange всегда должен менять state.
  • ✔ Для checkbox используйте checked вместо value .
  • ⚠ Не мутируйте state напрямую — используйте функцию обновления!

Лучшие практики

  • 💡 Держите все значения форм в state .
  • 💡 Выносите логику обновления в отдельные функции при сложных формах.
  • 💡 Используйте один state-объект для многих полей, если их много.
💡 Совет: Не смешивайте контролируемые и неконтролируемые поля в одной форме без необходимости.

Примеры контролируемых компонентов

Текстовое поле

Пояснение: 📝 Контролируемый input — значение всегда берется из state, обновляется через onChange. 

function NameInput() {
  const [name, setName] = React.useState("");
  return <input value={name} onChange={e => setName(e.target.value)} />;
}

Чекбокс

Пояснение: ☑ Для чекбокса контролируемым считается checked , не value ! 

function AgreeCheckbox() {
  const [agree, setAgree] = React.useState(false);
  return <input type="checkbox" checked={agree} onChange={e => setAgree(e.target.checked)} />;
}
