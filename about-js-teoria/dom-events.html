События в DOM: Введение

Событие — это любое действие пользователя (например, клик, ввод, прокрутка) или изменение на странице, на которое может реагировать JavaScript.
Обработка событий делает веб-страницы интерактивными и живыми!

  • Реагируй на действия пользователя: клики, ввод, движения мыши, нажатия клавиш.
  • Делай страницу динамичной: показывай сообщения, меняй элементы, запускай анимации.
  • Используй обработчики событий для создания современных интерфейсов.

Основы обработки событий

addEventListener — основной рекомендуемый способ отслеживания событий в JavaScript.
Позволяет добавить несколько обработчиков на одно событие, отделять логику от разметки и гибко управлять событиями.
Синтаксис:
element.addEventListener('тип_события', handler)

Пример: обработка клика по кнопке


const button = document.getElementById("my-button");
button.addEventListener("click", () => {
    alert("Кнопка нажата!");
});

Также можно использовать отдельную функцию-обработчик:


function handleClick() {
    console.log("Кнопка нажата!");
}
button.addEventListener("click", handleClick);

Удаление обработчика события


button.removeEventListener("click", handleClick);

Важно: Удалить обработчик можно только если он был добавлен как именованная функция!
Анонимные функции удалить невозможно, так как они не имеют ссылки.

Рекомендуется: всегда использовать addEventListener вместо HTML-атрибутов onclick , oninput и др.
Такой подход делает код более гибким и поддерживаемым.

Сравнение с HTML-атрибутами

Можно назначать обработчики событий прямо в HTML, например:
<button onclick="alert('Клик!')">Кнопка</button>
Но этот способ менее удобен для масштабирования и поддержки.

addEventListener позволяет:
— Назначать несколько обработчиков на одно событие
— Гибко управлять добавлением и удалением
— Работать с динамическими элементами

Популярные события

click - Клик мыши

Событие click срабатывает, когда пользователь кликает по элементу.

button.addEventListener("click", () => { ... }); 

	<button id="my-button">Нажми меня!</button>

	<script>
	document.getElementById("my-button").addEventListener("click", () => {
	   alert("Кнопка нажата!");
	});
	</script>

mouseover - Наведение курсора

Событие mouseover срабатывает, когда курсор мыши наводится на элемент.

element.addEventListener("mouseover", () => { ... }); 

	<div id="hover-box" style="width: 200px; height: 100px; background: #e0e7ff; border: 1px solid #cbd5e1; border-radius: 8px">Наведи курсор сюда</div>
	<div id="hover-result"></div>

	<script>
	document.getElementById("hover-box").addEventListener("mouseover", () => {
	   document.getElementById("hover-result").textContent = "Курсор наведен на элемент!";
	});
	</script>
Наведи курсор сюда

input - Ввод текста

Событие input срабатывает при вводе текста в поле ввода.

input.addEventListener("input", (e) => { ... }); 

<input id="text-input" type="text" placeholder="Введите текст">
<div id="input-result"></div>

<script>
document.getElementById("text-input").addEventListener("input", (e) => {
    document.getElementById("input-result").textContent = "Вы ввели: " + e.target.value;
});
</script>

keydown - Нажатие клавиши

Событие keydown срабатывает, когда пользователь нажимает клавишу на клавиатуре.

document.addEventListener("keydown", (e) => { ... }); 

<div id="keydown-result"></div>

<script>
document.addEventListener("keydown", (e) => {
    document.getElementById("keydown-result").textContent = "Вы нажали клавишу: " + e.key;
});
</script>

Важно: События могут всплывать вверх по DOM-дереву, поэтому можно ставить обработчики на родительские элементы.

Продвинутые техники работы с событиями

Для сложных задач — используем делегирование, фрагменты, наблюдатели и анимации!

Делегирование событий

document.addEventListener('click', (event) => {
  if (event.target.matches('.item')) {
    // ...
  }
});

Делегирование удобно для работы с динамическими списками или большим количеством элементов.

DocumentFragment

const fragment = document.createDocumentFragment();
const items = ['Один', 'Два', 'Три'];
items.forEach(item => {
  const li = document.createElement('li');
  li.textContent = item;
  fragment.appendChild(li);
});
document.getElementById('myList').appendChild(fragment);

MutationObserver

const observer = new MutationObserver((mutations) => {
  mutations.forEach(mutation => {
    console.log('Изменения в DOM:', mutation);
  });
});
observer.observe(document.body, { childList: true, subtree: true });

Оптимизация производительности

// Кэшируем элементы
const items = document.querySelectorAll('.item');
// requestAnimationFrame для плавной анимации
function animate() {
  // ...
  requestAnimationFrame(animate);
}

FAQ: Вопросы по событиям

  • Как удалить обработчик события?
    Используй removeEventListener с той же функцией, что и при добавлении.
  • Можно ли добавить несколько обработчиков одного события?
    Да, addEventListener позволяет добавить сколько угодно обработчиков.
  • Что такое делегирование событий?
    Это принцип, когда обработчик ставится на родитель, а реагирует на события дочерних элементов.
  • Для чего нужен MutationObserver?
    Для отслеживания изменений в DOM — добавление, удаление, изменение узлов.
  • Как избежать утечек памяти?
    Удаляй ненужные обработчики, не вешай их на слишком большое количество элементов.
