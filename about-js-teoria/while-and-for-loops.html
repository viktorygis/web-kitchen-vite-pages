Циклы в JavaScript

Циклы позволяют повторять выполнение блока кода, пока выполняется некоторое условие, или определённое число раз. Используются для перебора коллекций, повторных вычислений и автоматизации однотипных операций.

Основные типы циклов (базовые):

  • while — повторяет, пока условие истинно (проверка до итерации).
  • do...while — выполняет тело как минимум один раз (проверка после итерации).
  • for — универсальная форма с явным началом / условием / шагом.

Также есть for...of , for...in — рассматриваются отдельно (в темах про массивы и объекты).

Основные понятия цикла

  • Итерация — одно выполнение тела цикла.
  • Условие — выражение, определяющее продолжать или закончить цикл.
  • Счётчик — переменная, изменяющаяся на каждой итерации (например, i++ ).
  • Тело цикла — код, который повторяется.
  • Выход — момент, когда условие перестаёт быть истиной или выполнен break .

Общий принцип:

Инициализация → Проверка условия → (true?) Выполнение тела → Шаг → Проверка → ... → (false?) Выход

while — цикл с предусловием

Цикл while — это «повторяй, пока условие истинно». Он особенно полезен, когда ты не знаешь заранее, сколько итераций будет, но точно знаешь, при каком условии нужно остановиться.

Синтаксис:

while (условие) {
  // тело цикла
}

Базовый пример

let i = 0;              // 1. Инициализация счётчика
while (i < 7) {          // 2. Проверка условия перед каждой итерацией
  console.log("Повторение:", i); // 3. Тело (выполняется, если условие true)
  i++;                  // 4. Шаг — обязательно, иначе бесконечный цикл
}

Что произойдёт в консоли:

Повторение: 0
Повторение: 1
Повторение: 2
Повторение: 3
Повторение: 4
Повторение: 5
Повторение: 6

Число 7 не выводится, потому что условие i < 7 становится ложным при i === 7 .

Разбор по шагам

  1. Инициализация: i = 0 .
  2. Проверка условия: i < 7 → true → выполняем тело.
  3. Выполнение тела: вывод "Повторение: 0" .
  4. Изменение счётчика: i++ → i = 1 .
  5. Повтор: снова проверяем условие — пока true, цикл продолжается.
  6. Завершение: при i = 7 условие false → выходим.

Когда использовать

  • Количество повторов заранее неизвестно .
  • Ожидаешь «сигнал остановки» (например, ввод пустой строки).
  • Нужна реакция на внешние изменения состояния.

Пример с пользовательским вводом

let line;
while ((line = prompt("Введите что-нибудь (пусто — выход)")) !== "") {
  console.log("Введено:", line);
}
console.log("Завершено.");

Типичные ошибки

  • Нет изменения счётчика: условие никогда не станет ложным ⇒ бесконечный цикл.
  • Слишком сложное условие: вынеси части во временные переменные.
  • Неожиданное приведение типов: например, сравнение строки с числом ( "7" < 10 → true).

Сравнение с for

for while
Все части в одной строке Логика распределена по блокам
Хорош, когда известен диапазон Хорош, когда неизвестно число итераций
Счётчик очевиден Можно обойтись без явного счётчика

Расширенные примеры

// Бесконечный цикл с ручным break
let n = 0;
while (true) {
  if (n === 3) break;
  console.log(n);
  n++;
}

// Отслеживание состояния
let attempts = 0;
let success = false;
while (!success && attempts < 5) {
  attempts++;
  success = Math.random() > 0.7;
}
console.log("Успешно?", success, "Попыток:", attempts);

do...while — цикл с постусловием

Цикл do...while похож на while , но есть ключевое отличие:
тело выполняется минимум один раз , даже если условие изначально ложно. Полезен, когда нужно сначала что-то сделать (показать меню, запросить ввод), а уже потом решать — повторять ли.

Синтаксис:

do {
  // тело цикла (выполнится хотя бы один раз)
} while (условие);

Базовый пример

let i = 0;
do {
  console.log("Повторение:", i);
  i++;
} while (i < 5);

Вывод в консоль:

Повторение: 0
Повторение: 1
Повторение: 2
Повторение: 3
Повторение: 4

Когда i становится равным 5 — условие i < 5 ложно, цикл завершается.

Разбор по шагам

  1. Инициализация: i = 0 .
  2. Первое выполнение тела: выводит Повторение: 0 .
  3. Шаг: i++ → i = 1 .
  4. Проверка условия: i < 5 → true → вторая итерация.
  5. … повтор до тех пор, пока условие не станет false.
  6. Завершение: при i = 5 условие false → выход.

Почему «хотя бы один раз»?

let n = 10;
do {
  console.log("Это выведется даже при ложном условии");
} while (n < 0);

Здесь тело выполнится один раз, хотя n < 0 уже изначально false.

Практический пример (валидация ввода)

let value;
do {
  value = prompt("Введите число больше 10 (Отмена - выход)", "");
  if (value === null) {
    console.log("Отменено пользователем.");
    break;
  }
} while (+value <= 10);

if (value !== null) {
  console.log("Вы ввели:", value);
}

Когда использовать:

  • Нужно выполнить действие хотя бы один раз (меню / ввод / первичная инициализация).
  • Проверка условия логически следует после выполнения шага.
  • Условие зависит от результата тела (например, пользовательского ввода).

Сравнение с while

Характеристика while do...while
Порядок Сначала проверка, потом тело Сначала тело, потом проверка
Минимум итераций 0 1
Подходит для Когда возможно «ни разу» Гарантированная первичная операция
Читаемость Привычнее / чаще используется Реже, но уместен в сценариях ввода

Типичные ошибки

  • Забыл выход: условие всегда true ⇒ бесконечный цикл.
  • Неправильное условие: проверяется после тела — иногда «лишняя» итерация.
  • Преобразование типов: prompt() возвращает строку — используй +value или Number(value) для сравнения.

Расширенные примеры

// 1. Повторяем до случайного успеха
let attempts = 0;
let ok;
do {
  attempts++;
  ok = Math.random() > 0.75;
} while (!ok && attempts < 10);
console.log("Попыток:", attempts, "Успех?", ok);

// 2. Минимум одно вычисление
let cache;
do {
  cache = computeHeavy();
} while (!cache.valid);

// (условная функция)
function computeHeavy() {
  return { valid: Math.random() > 0.3 };
}

for — универсальный цикл

Цикл for — это «компактный повторитель»: запускает один и тот же блок кода заданное число раз. Вся логика (старт, условие, изменение счётчика) сосредоточена в одной строке.

Структура цикла

for (инициализация; условие; шаг) {
  // тело цикла
}
  • Инициализация — выполняется один раз перед первой проверкой (обычно объявление счётчика: let i = 0 ).
  • Условие — проверяется перед каждой итерацией. Если false — выходим.
  • Шаг — выполняется после тела каждой итерации (часто i++ ).
  • Тело — код, который повторяется.

Разбор базового примера

for (let i = 0; i < 5; i++) {
  console.log("Повторение:", i);
}

Что происходит по шагам:

  1. let i = 0 — инициализация (один раз).
  2. Проверка i < 5 → true → выполняем тело → вывод Повторение: 0 .
  3. i++ → i = 1 , снова проверка условия → и так далее.
  4. Когда i становится 5, условие i < 5 ложно → цикл завершается.
Вывод в консоль:
Повторение: 0
Повторение: 1
Повторение: 2
Повторение: 3
Повторение: 4

Части цикла (сводная таблица)

Часть Когда выполняется Пример
Инициализация Перед первой проверкой условия let i = 0
Условие Перед каждой итерацией i < 5
Тело Если условие true console.log(i)
Шаг После тела, перед следующей проверкой i++

Варианты и гибкость

// 1. Пропуск инициализации (счётчик объявлен заранее)
let k = 0;
for (; k < 3; k++) console.log(k);

// 2. Пропуск шага
for (let j = 0; j < 3;) {
  console.log(j);
  j++;
}

// 3. Несколько переменных
for (let a = 0, b = 5; a < b; a++, b--) {
  console.log(a, b);
}

// 4. Обратный отсчёт
for (let n = 5; n > 0; n--) {
  console.log(n);
}

// 5. Бесконечный цикл (нужен break)
for (;;) {
  break;
}

Когда выбирать for :

  • Заранее известно количество повторов.
  • Нужен индекс (позиция в массиве).
  • Нужно компактно описать управление счётчиком.

Типичные ошибки

  • Бесконечный цикл: забыли изменить счётчик ( i++ ).
  • Off-by-one: i <= arr.length вместо i < arr.length .
  • Неподходящий тип объявлен: использование var вместо let (протекает за пределы цикла).
  • Лишний индекс: когда индекс не нужен — рассмотрите for...of .

// Пропуск чётных

// Преждевременный выход
for (let i = 0; i < 100; i++) {

// Агрегация

for (let i = 1; i <= 5; i++) {
  sum += i;

console.log("Сумма 1..5 =", sum); // 15

Гибкость цикла for

Любую часть можно опустить.

// Без "начала"
let i = 0;
for (; i < 3; i++) console.log(i);

// Без "шага"
for (let j = 0; j < 3;) {
  console.log(j);
  j++;
}

// Бесконечный цикл (нужен break внутри)
for (;;) {
  break;
}

Когда НЕ использовать for:

  • Для перебора массива ради обработки каждого элемента → лучше for...of (выше читаемость).
  • Для поиска / фильтрации → чаще подойдут Array.prototype.filter / find / some / every .

Управление выполнением цикла: break , continue и бесконечные циклы

Внутри цикла можно управлять потоком выполнения: досрочно прервать цикл с помощью break , пропустить текущую итерацию через continue , а также — осознанно построить «бесконечный» цикл с ручким выходом.

Бесконечный цикл

Бесконечный цикл — это цикл, у которого условие завершения никогда не становится ложным (или вообще отсутствует). Он полезен, когда требуется «постоянный цикл ожидания» с явным выходом внутри. 

while (true) {
  console.log("Этот цикл никогда не закончится сам!");
  // Нужен break (или return) внутри при определённом условии
}
Никогда не оставляй бесконечный цикл без выхода — это «повисание» интерфейса / нагрузки на CPU.

Как избежать непреднамеренного бесконечного цикла:

  • Изменяй состояние: счётчик / данные должны двигать условие к завершению.
  • Проверяй условие: может ли оно стать false вообще?
  • Используй break : когда наступает «условие выхода».
  • Ограничивай попытки: добавь «страховку» (макс. число итераций). 
// Осознанный бесконечный цикл с защитой
let attempts = 0;
while (true) {
  attempts++;
  if (Math.random() > 0.85) {
    console.log("Успех на попытке", attempts);
    break;
  }
  if (attempts > 1000) {
    console.log("Принудительный выход (лимит)");
    break;
  }
}

break

break немедленно завершает текущий цикл (или внешний по метке). 

for (let i = 0; i < 10; i++) {
  if (i === 5) {
    console.log("Нашли 5 — выходим");
    break;
  }
}

Применение:

  • Поиск первого подходящего элемента.
  • Прерывание по «сигналу» (ошибка, флаг, внешнее событие).
  • Ограничение глубины / времени.

continue

continue пропускает остаток текущей итерации и переходит к следующей проверке условия. 

for (let i = 0; i < 10; i++) {
  if (i % 2 === 0) continue; // пропускаем чётные
  console.log("Нечётное:", i);
}
Совет: Частые continue иногда ухудшают читаемость — попробуй инвертировать условие: 
// Вместо:
for (let x of arr) {
  if (!x.active) continue;
  process(x);
}

// Можно:
for (let x of arr) {
  if (x.active) {
    process(x);
  }
}

Сравнение break и continue

Оператор Действие Типичный сценарий
break Полностью завершает цикл Найден результат / достигнут лимит
continue Пропускает оставшуюся часть тела итерации Пропустить неинтересные элементы

Вложенные циклы и досрочный выход

outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 2) {
      console.log("Выходим из ОБОИХ циклов");
      break outer; // метка
    }
  }
}
Метки используй экономно — иногда лучше вынести внутренний цикл в функцию и сделать return .

Типичные ошибки

  • Забыли шаг: счётчик не меняется → цикл бесконечен.
  • Условие всегда true: неверная логика ( i <= length при инкременте и неизменном length).
  • break вне цикла: синтаксическая ошибка.
  • Продолжение после цели: забыли break и зря тратим время.
  • Злоупотребление continue: «рваный» поток чтения.

Паттерны с безопасным выходом

// 1. Лимит попыток
let ok = false;
for (let tries = 0; tries < 5; tries++) {
  ok = Math.random() > 0.6;
  if (ok) { console.log("Успех на", tries + 1); break; }
}

// 2. break вместо флага
// Антипаттерн:
let found = false;
for (const x of list) {
  if (x.id === target) { found = true; }
}
if (found) console.log("Нашли");

// Лучше:
for (const x of list) {
  if (x.id === target) {
    console.log("Нашли");
    break;
  }
}

Мини‑практика

💡 Совет: если ловишь себя на множестве continue / break — возможно, стоит пересмотреть структуру кода или использовать методы массива ( some , every , find ).

Метки (labels) для вложенных циклов

Позволяют выйти из внешнего цикла изнутри внутреннего.

outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    const val = prompt(`(${i},${j})`, "");
    if (val === null) break outer;
  }
}
console.log("Готово");
Используй редко: метки могут сделать поток управления запутанным. Часто их можно заменить флагами или вынесением логики в функцию.

Сравнение типов циклов

Цикл Когда использовать Проверка условия Мин. итераций Пример сценария
while Неизвестно число повторов заранее Перед каждой итерацией 0 Чтение до пустой строки
do...while Нужно выполнить минимум 1 раз После итерации 1 Меню ввода, валидация
for Число повторов известно / индекс важен Перед каждой итерацией 0 Перебор первых N элементов

Лучшие практики

  • Избегай «магических чисел» — выноси пределы в переменные/константы.
  • Используй for...of для итерации по массивам, когда индекс не нужен.
  • Используй методы массива ( map , filter , reduce ) для декларативной логики.
  • Преждевременный break вместо лишних флагов увеличивает читаемость.
  • Не изменяй длину массива (например, push / splice ) внутри обычного for , если это влияет на индексы.
  • Старайся, чтобы тело цикла было небольшим — вынеси сложную логику в функцию.

Ловушки

  • Бесконечный цикл: забыли изменить счётчик или неверное условие ( while (true) без break ).
  • Off-by-one ошибка: i <= arr.length вместо i < arr.length .
  • Повторное объявление счетчика: использование var вместо let может привести к утечкам в область видимости.
  • Ненужный continue: можно переписать условие в позитивной форме.
  • Модификация массива при проходе: меняет индексы — лучше итерироваться с конца или создавать копию.
  • Использование for...in для массива: приводит к перебору «лишних» ключей / не гарантирует порядок.

Итоги

  • while — когда заранее неизвестно число итераций.
  • do...while — гарантированно одна итерация.
  • for — когда нужно явное управление счётчиком.
  • break и continue управляют потоком внутри цикла.
  • Метки нужны редко — используем осознанно.
  • Современные методы массивов часто читаемее ручных циклов.
