Введение Sass-переменные CSS-переменные Сравнение Рекомендации

Разница между Sass-переменными и CSS-переменными

Переменные в CSS — это мощный инструмент для создания поддерживаемых и гибких стилей. Однако между Sass-переменными (препроцессор) и CSS-переменными (нативные кастомные свойства) есть принципиальные различия.

Ключевые отличия:

  • Sass-переменные компилируются в CSS
  • CSS-переменные существуют в браузере
  • Sass имеет расширенные функции для работы с переменными
  • CSS-переменные можно изменять через JavaScript
  • Sass-переменные работают во всех браузерах

Основные термины:

  • Sass-переменные - переменные препроцессора Sass
  • CSS-переменные - кастомные свойства CSS (var(--name))
  • Препроцессор - инструмент для расширения возможностей CSS
  • Компиляция - преобразование Sass в обычный CSS
  • Каскадирование - наследование свойств в CSS

Эксперимент: Просмотр скомпилированного кода

Посмотрите, как Sass-переменные преобразуются в обычный CSS:

Исходный Sass

$primary-color: #3498db;
$padding: 1rem;

.button {
  background: $primary-color;
  padding: $padding;
}

Скомпилированный CSS

.button {
  background: #3498db;
  padding: 1rem;
}

Sass-переменные

Особенности Sass-переменных

  • Определяются с помощью символа $
  • Компилируются в фиксированные значения
  • Доступны только на этапе компиляции
  • Не существуют в итоговом CSS
  • Поддерживают сложные операции и функции
// Определение переменных
$primary-color: #3498db;
$base-padding: 1rem;
$border-radius: 4px;

// Использование
.button {
  background: $primary-color;
  padding: $base-padding * 1.5;
  border-radius: $border-radius;
}

Преимущества Sass

  • Математические операции - можно выполнять вычисления с переменными
  • Функции для работы с цветами - darken(), lighten(), transparentize()
  • Миксины и функции - создание переиспользуемых блоков кода
  • Полная совместимость - работает во всех браузерах после компиляции
  • Логика и циклы - условные операторы и циклы для генерации стилей 
// Пример использования функций
$primary: #3498db;

.button {
  background: $primary;
  &:hover {
    background: darken($primary, 10%);
  }
}

// Пример цикла
@for $i from 1 through 5 {
  .mt-#{$i} {
    margin-top: #{$i * 0.5}rem;
  }
}

CSS-переменные (кастомные свойства)

Особенности CSS-переменных

  • Определяются с помощью --имя-переменной
  • Используются с функцией var()
  • Существуют в браузере
  • Могут изменяться во время выполнения
  • Поддерживают каскадирование и наследование
/* Определение переменных */
:root {
  --primary-color: #3498db;
  --base-padding: 1rem;
  --border-radius: 4px;
}

/* Использование */
.button {
  background: var(--primary-color);
  padding: var(--base-padding);
  border-radius: var(--border-radius);
}

Преимущества CSS-переменных

  • Динамическое изменение - можно менять через JavaScript
  • Каскадирование - переменные наследуются как обычные свойства
  • Темы оформления - легкое переключение между темами
  • Доступ в медиа-запросах - изменение значений на разных разрешениях
  • Нативные в браузере - не требуют компиляции 
/* Переключение тем */
:root {
  --bg-color: white;
  --text-color: black;
}

[data-theme="dark"] {
  --bg-color: #333;
  --text-color: white;
}

body {
  background: var(--bg-color);
  color: var(--text-color);
}

/* Изменение через JavaScript */
document.documentElement.style.setProperty('--bg-color', '#222');

Сравнение возможностей

Характеристика Sass-переменные CSS-переменные

Синтаксис $variable: value; --variable: value;

Функции для работы с цветами Да (darken, lighten и др.) Нет

Когда что использовать?

Используйте Sass-переменные:

  • Для сложных вычислений и операций
  • Когда нужны функции для работы с цветами
  • Для генерации стилей через циклы
  • В проектах с поддержкой старых браузеров
  • Для конфигурации миксинов и функций

Используйте CSS-переменные:

  • Для динамического изменения стилей
  • При реализации смены тем оформления
  • Для значений, зависящих от размера экрана
  • Когда нужно изменять стили через JavaScript
  • Для каскадного наследования значений

Рекомендации и лучшие практики

Комбинированный подход

Оптимальное решение - использовать оба типа переменных вместе, где это уместно:

// Sass-переменные для сложных вычислений
$base-spacing: 1rem;
$primary-color: #3498db;

:root {
  // Экспорт Sass-переменных в CSS-переменные
  --spacing: #{$base-spacing};
  --primary: #{$primary-color};
  --primary-dark: #{darken($primary-color, 10%)};
}

.element {
  // Использование CSS-переменных
  padding: calc(var(--spacing) * 2);
  background: var(--primary);

  &:hover {
    background: var(--primary-dark);
  }
}

Практические советы

  • Используйте префиксы для CSS-переменных (--theme-primary, --spacing-md)
  • Определяйте CSS-переменные в :root для глобального доступа
  • Задавайте значения по умолчанию с помощью var(--var, default)
  • Используйте Sass для сложных вычислений перед экспортом в CSS-переменные
  • Проверяйте поддержку браузеров для CSS-переменных 
/* Значение по умолчанию */
.element {
  background: var(--theme-bg, #f0f0f0);
}

/* Проверка поддержки в CSS */
@supports (--css: variables) {
  /* Стили с CSS-переменными */
}

/* Проверка поддержки в JavaScript */
if (window.CSS && window.CSS.supports && window.CSS.supports('--a', 0)) {
  // Поддерживает CSS-переменные
}
